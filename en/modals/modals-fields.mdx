## Field types overview

Spraxium modals support 11 field types, each declared through a property-level decorator on your _@ModalComponent_ class. Text inputs and selects are the most common, but the system also includes radio groups, checkboxes, file uploads, and read-only text displays. Every field decorator accepts a configuration object with properties specific to the field type, plus shared properties like _label_ and _required_.

The field order in the rendered modal matches the declaration order of the decorated properties in the class. Discord modals have a maximum of 5 action rows, and each field occupies one row. This limit means a single modal can contain at most 5 fields, though dynamic and conditional fields can alter what appears for each user.

## Text inputs

The _@ModalInput_ decorator creates a text input field. The _style_ property controls whether the field renders as a single-line (_short_) or multi-line (_paragraph_) input. Short fields are ideal for names, subjects, and brief values. Paragraph fields suit descriptions, messages, and longer content.

```typescript filename="src/components/report.modal.ts"
import { ModalComponent, ModalInput } from "@spraxium/components";

@ModalComponent({ id: "report-modal", title: "Submit Report" })
export class ReportModal {
  @ModalInput({
    label: "Title",
    placeholder: "A short title for the report",
    required: true,
    minLength: 5,
    maxLength: 100,
    style: "short",
  })
  title!: string;

  @ModalInput({
    label: "Description",
    placeholder: "Describe the issue in detail...",
    required: true,
    minLength: 20,
    maxLength: 4000,
    style: "paragraph",
  })
  description!: string;

  @ModalInput({
    label: "Reproduction Steps",
    placeholder: "Optional steps to reproduce",
    required: false,
    style: "paragraph",
    value: "1. \n2. \n3. ",
  })
  steps!: string;
}
```

## String selects with choices

The _@ModalSelect_ decorator creates a dropdown select menu inside the modal. Define the available options using the _@ModalChoice_ stackable decorator on the same property. Each choice has a _label_ (displayed to the user), _value_ (sent on submission), and an optional _description_.

```typescript filename="src/components/category-select.modal.ts"
import { ModalComponent, ModalSelect, ModalChoice } from "@spraxium/components";

@ModalComponent({ id: "ticket-modal", title: "Create Ticket" })
export class TicketModal {
  @ModalSelect({
    label: "Category",
    placeholder: "Select a ticket category",
    required: true,
  })
  @ModalChoice({
    label: "Bug Report",
    value: "bug",
    description: "Something is broken",
  })
  @ModalChoice({
    label: "Feature Request",
    value: "feature",
    description: "Suggest an improvement",
  })
  @ModalChoice({
    label: "General Question",
    value: "question",
    description: "Ask a question",
  })
  category!: string;

  @ModalInput({
    label: "Details",
    style: "paragraph",
    required: true,
  })
  details!: string;
}
```

## Entity selects

Entity select decorators let users pick server members, roles, channels, or mentionable entities directly from the modal. These fields use Discord's native entity pickers, so they show avatars, role colors, and channel icons. No _@ModalChoice_ decorators are needed because the options are populated from the server.

<CodeTabs>
<Tab label="User Select">
```typescript filename="src/components/assign.modal.ts"
import { ModalComponent, ModalUserSelect } from '@spraxium/components';

@ModalComponent({ id: 'assign-modal', title: 'Assign User' })
export class AssignModal {
@ModalUserSelect({
label: 'Assignee', placeholder: 'Pick a team member', required: true, })
assignee!: string;
}

````
</Tab>
<Tab label="Channel Select">
```typescript filename="src/components/move-channel.modal.ts"
import { ModalComponent, ModalChannelSelect } from '@spraxium/components';
import { ChannelType } from 'discord.js';

@ModalComponent({ id: 'move-modal', title: 'Move to Channel' })
export class MoveChannelModal {
  @ModalChannelSelect({
    label: 'Target Channel', placeholder: 'Select a text channel', required: true, channelTypes: [ChannelType.GuildText], })
  channel!: string;
}
````

</Tab>

## Radio groups and checkboxes

Radio groups present mutually exclusive options where the user must pick exactly one. Checkbox groups allow multiple selections. A single *@ModalCheckbox* renders as a boolean toggle. All choice-based fields use *@ModalOption* (an alias for *@ModalChoice*) to define their options.

<CodeTabs>
<Tab label="Radio Group">
```typescript filename="src/components/priority.modal.ts"
import { ModalComponent, ModalRadioGroup, ModalOption } from '@spraxium/components';

@ModalComponent({ id: 'priority-modal', title: 'Set Priority' })
export class PriorityModal {
  @ModalRadioGroup({ label: 'Priority Level' })
  @ModalOption({ label: 'Low', value: 'low' })
  @ModalOption({ label: 'Medium', value: 'medium' })
  @ModalOption({ label: 'High', value: 'high' })
  @ModalOption({ label: 'Critical', value: 'critical' })
  priority!: string;
}
````

</Tab>
<Tab label="Checkbox Group">
```typescript filename="src/components/notifications.modal.ts"
import { ModalComponent, ModalCheckboxGroup, ModalOption } from '@spraxium/components';

@ModalComponent({ id: 'notif-modal', title: 'Notification Settings' })
export class NotificationsModal {
@ModalCheckboxGroup({
label: 'Notification Types', minValues: 1, maxValues: 3, })
@ModalOption({ label: 'Mentions', value: 'mentions' })
@ModalOption({ label: 'Direct Messages', value: 'dms' })
@ModalOption({ label: 'Server Events', value: 'events' })
types!: string[];
}

````
</Tab>
<Tab label="Single Checkbox">

  signature!: string;

## File uploads and text displays

The _@ModalFileUpload_ decorator adds an attachment field to the modal. The _@ModalTextDisplay_ decorator adds read-only static text that is not submitted as a field value. Text displays are useful for showing instructions, warnings, or context information within the modal form itself.

```typescript filename="src/components/upload-evidence.modal.ts"
import {
  ModalComponent,
  ModalTextDisplay,
  ModalFileUpload,
  ModalInput,
} from "@spraxium/components";

@ModalTextDisplay({
  content:
    "Please upload a screenshot or log file as evidence. Accepted formats: PNG, JPG, TXT.",
})
@ModalComponent({ id: "evidence-modal", title: "Upload Evidence" })
export class UploadEvidenceModal {
  @ModalFileUpload({
    label: "Evidence File",
    required: true,
  })
  file!: string;

  @ModalInput({
    label: "Additional Notes",
    style: "paragraph",
    required: false,
    placeholder: "Any extra context about the evidence...",
  })
  notes!: string;
}
```

```typescript filename="src/utils/modal-helpers.ts"
import { Modal } from "@spraxium/components";

export function createReasonField(required = true) {
  return Modal.field.text({
    id: "reason",
    label: "Reason",
    placeholder: "Provide a reason...",
    style: "paragraph",
    required,
    minLength: 10,
  });
}

export function createPriorityChoices() {
  return [
    Modal.choice({ label: "Low", value: "low" }),
    Modal.choice({ label: "Medium", value: "medium" }),
    Modal.choice({ label: "High", value: "high" }),
  ];
}
```
