## Button types

Spraxium provides two button variants, each with its own decorator and use case. Static buttons have a fixed _customId_ and fire a handler when clicked. Link buttons open a URL in the user's browser and do not fire any handler.

Both types are defined as decorated classes and rendered through _ButtonService_. The visual properties, label, style, emoji, and disabled state, are configured in the decorator and stored as metadata. The renderer reads this metadata at build time and produces the appropriate discord.js _ButtonBuilder_ instance.

## Static buttons

A static button is the simplest interactive component. You decorate a class with _@Button_, providing a _customId_ and a _label_ at minimum. The _customId_ must be unique across your entire bot because the dispatcher matches incoming interactions by exact _customId_ equality. The _style_ field maps to Discord's button styles and defaults to `'primary'` if omitted.

<CodeTabs>
  <Tab label="Component" language="typescript">
```typescript filename="src/components/buttons/confirm.button.ts"
import { Button } from '@spraxium/components';

@Button({
customId: 'confirm_action', label: 'Confirm', style: 'success', emoji: '✅', })
export class ConfirmButton {}

````
  </Tab>
  <Tab label="Handler" language="typescript">
```typescript filename="src/components/buttons/confirm.handler.ts"
import { ButtonHandler, Ctx } from '@spraxium/components';
import type { ButtonInteraction } from 'discord.js';
import { ConfirmButton } from './confirm.button';

@ButtonHandler(ConfirmButton)
export class ConfirmButtonHandler {
  async handle(@Ctx() interaction: ButtonInteraction): Promise<void> {
    await interaction.reply({ content: 'Action confirmed.', ephemeral: true });
  }
}
````

  </Tab>
</CodeTabs>

The handler class must be decorated with _@ButtonHandler_ and pass the component class as its argument. This creates a metadata link that the dispatcher resolves during bootstrap. The _handle_ method receives parameters through decorators: _@Ctx()_ injects the raw _ButtonInteraction_, and _@FlowContext()_ injects the context object when present.

## Link buttons

Link buttons open a URL in the user's browser without firing any handler on the bot side. They are useful for directing users to documentation, dashboards, or external tools. Because Discord requires a different button style for links, the _@LinkButton_ decorator omits _customId_ and _style_ in favor of a _url_ field.

```typescript filename="src/components/buttons/docs-link.button.ts"
import { LinkButton } from "@spraxium/components";

@LinkButton({
  url: "https://docs.example.com",
  label: "View Documentation",
  emoji: "📖",
})
export class DocsLinkButton {}
```

Link buttons can be mixed with static buttons in the same action row. You pass all button classes to _ButtonService.build_ and the renderer produces the correct builder type for each one based on the metadata.

## Rendering with ButtonService

_ButtonService_ is the injectable service that transforms decorated classes into discord.js action rows. It provides two methods for different rendering scenarios. All methods read component metadata via reflection, so the service does not need explicit configuration beyond the decorated classes.

<Table>
  <TableHead>
    <TableRow>
      <TableCell header>Method</TableCell>
      <TableCell header>Signature</TableCell>
      <TableCell header>Purpose</TableCell>
    </TableRow>
  </TableHead>
  <TableBody>
    <TableRow>
      <TableCell>*build*</TableCell>
      <TableCell>`build(input: Class | Class[], context?): ActionRowBuilder`</TableCell>
      <TableCell>Renders one or more static/link buttons into a single action row. Attaches context to all buttons when provided.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>*rowWithContext*</TableCell>
      <TableCell>`rowWithContext(context, ...Classes): ActionRowBuilder`</TableCell>
      <TableCell>Convenience wrapper that calls *build* with context as the first argument and spreads classes.</TableCell>
    </TableRow>

  </TableBody>
</Table>

<Callout type="tip">
  Discord limits action rows to five buttons. The service throws an error if you
  pass more than five classes to *build*. For layouts with more buttons, create
  multiple rows by calling *build* multiple times and pass each row individually
  in the *components* array.
</Callout>

```typescript filename="src/commands/moderation.command-handler.ts"
import { Ctx, SlashCommandHandler } from "@spraxium/common";
import { ButtonService, ContextService } from "@spraxium/components";
import type { ChatInputCommandInteraction } from "discord.js";
import { ConfirmButton } from "../components/buttons/confirm.button";
import { CancelButton } from "../components/buttons/cancel.button";
import { DocsLinkButton } from "../components/buttons/docs-link.button";
import { ModerationCommand } from "./moderation.command";

@SlashCommandHandler(ModerationCommand)
export class ModerationCommandHandler {
  constructor(
    private readonly buttons: ButtonService,
    private readonly context: ContextService,
  ) {}

  async handle(@Ctx() interaction: ChatInputCommandInteraction): Promise<void> {
    const ctx = await this.context.create({ targetUserId: "123456" });
    const actionRow = this.buttons.rowWithContext(
      ctx,
      ConfirmButton,
      CancelButton,
    );
    const linkRow = this.buttons.build(DocsLinkButton);

    await interaction.reply({
      content: "Review the moderation action below.",
      components: [actionRow, linkRow],
    });
  }
}
```

## Module registration

Only handler classes need to be registered, and they go in the _handlers_ array of your feature module. Component classes decorated with _@Button_ or _@LinkButton_ are metadata-only and are never added to any module. The dispatcher reads their customId and configuration directly from the class reference stored inside each _@ButtonHandler_ decorator, with no DI instantiation required.

```typescript filename="src/modules/moderation/moderation.module.ts"
import { Module } from "@spraxium/common";
import { ConfirmButtonHandler } from "./components/confirm.handler";
import { CancelButtonHandler } from "./components/cancel.handler";
import { UserBanButtonHandler } from "./components/user-ban.handler";

@Module({
  handlers: [ConfirmButtonHandler, CancelButtonHandler, UserBanButtonHandler],
})
export class ModerationModule {}
```
