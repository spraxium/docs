## The dev block

The _dev_ block controls the local development watcher that restarts your bot process when source files change. All fields are optional. The default entrypoint is `src/main.ts` and the default debounce is 300 milliseconds; most projects only need to adjust _debounce_ to prevent restart storms during rapid saves.

```typescript filename="spraxium.config.ts"
import { defineConfig } from "@spraxium/core";

export default defineConfig({
  dev: {
    entrypoint: "src/main.ts",
    debounce: 500,
    include: ["locales"],
    exclude: [".spraxium/**", "dist/**"],
  },
});
```

<Callout tone="warning" title="Exclude generated directories">
  Always add .spraxium/** and dist/** to the *exclude* list. These directories
  change during build operations, and watching them causes infinite restart
  loops that destabilize the watcher process.
</Callout>

## Environment resolution

`NODE_ENV` is read once during startup and mapped to a _ConfigEnv_ object. This value never changes after the config has been loaded, so all environment-based branching must happen inside your _defineConfig_ callback.
