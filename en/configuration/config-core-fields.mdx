## debug

The _debug_ flag enables extra diagnostic output across every subsystem. When active, the framework emits additional trace lines during module loading, provider resolution, command registration, and interaction dispatch. This output is intended for troubleshooting and should not stay active in high-throughput production deployments.

The flag accepts a plain boolean or any _ConfigEnv_ expression, so you can scope verbosity to a specific environment.

## logger

The _logger_ block controls output policy for the built-in logger. It does not affect business logic. The most commonly configured options are _maskTokens_ (replaces token-like strings with a safe placeholder), _commandLogging_ (emits a trace line on every command dispatch), and _discord_ (sends selected log levels to a webhook channel).

```typescript filename="config/logger.config.ts"
import type { SpraxiumLoggerConfig } from "@spraxium/core";

export const loggerConfig: SpraxiumLoggerConfig = {
  maskTokens: true,
  commandLogging: true,
  discord: {
    type: "channel",
    webhookUrl: process.env.LOGGER_WEBHOOK!,
    levels: ["error", "warn"],
    channelId: "1462519588775395429",
  },
};
```

## plugins

The _plugins_ array collects entries returned by plugin factories. Each entry carries a _namespace_ string and a typed config payload; the framework uses the namespace to route config to the correct plugin at runtime. Plugin factories and first-party options are covered on the Plugin Registration page.
