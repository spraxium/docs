## debug

The *debug* flag enables extra diagnostic output across every subsystem. When active, the framework emits additional trace lines during module loading, provider resolution, command registration, and interaction dispatch. This output is intended for troubleshooting and should not stay active in high-throughput production deployments.

The flag accepts a plain boolean or any *ConfigEnv* expression, so you can scope verbosity to a specific environment.

## logger

The *logger* block controls output policy for the built-in logger. It does not affect business logic. The most commonly configured options are *maskTokens* (replaces token-like strings with a safe placeholder), *commandLogging* (emits a trace line on every command dispatch), and *discord* (sends selected log levels to a webhook channel).

```typescript filename="config/logger.config.ts"
import type { SpraxiumLoggerConfig } from '@spraxium/core';

export const loggerConfig: SpraxiumLoggerConfig = {
  maskTokens: true, commandLogging: true, discord: {
    type: 'channel', webhookUrl: process.env.LOGGER_WEBHOOK!, levels: ['error', 'warn'], channelId: '1462519588775395429', }, };
```

## plugins

The *plugins* array collects entries returned by plugin factories. Each entry carries a *namespace* string and a typed config payload; the framework uses the namespace to route config to the correct plugin at runtime. Plugin factories and first-party options are covered on the Plugin Registration page.
