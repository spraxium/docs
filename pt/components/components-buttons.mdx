## Tipos de botão

O Spraxium oferece duas variantes de botão, cada uma com seu próprio decorator e caso de uso. Botões estáticos têm um _customId_ fixo e disparam um handler quando clicados. Botões de link abrem uma URL no navegador do usuário e não disparam nenhum handler.

Ambos os tipos são definidos como classes decoradas e renderizados através do _ButtonService_. As propriedades visuais, como label, estilo, emoji e estado desabilitado, são configuradas no decorator e armazenadas como metadados. O renderizador lê esses metadados no momento do build e produz a instância apropriada do _ButtonBuilder_ do discord.js.

## Botões estáticos

Um botão estático é o componente interativo mais simples. Você decora uma classe com _@Button_, fornecendo um _customId_ e um _label_ no mínimo. O _customId_ deve ser único em todo o seu bot porque o dispatcher faz a correspondência de interações por igualdade exata de _customId_. O campo _style_ mapeia para os estilos de botão do Discord e usa `'primary'` como padrão se omitido.

<CodeTabs>
<Tab label="Componente">
```typescript filename="src/components/confirm.button.ts"
import { Button } from '@spraxium/components';

@Button({
customId: 'confirm-action', label: 'Confirm', style: 'success', emoji: '✅', })
export class ConfirmButton {}

````
</Tab>
<Tab label="Handler">
```typescript filename="src/components/confirm.handler.ts"
import { ButtonHandler, Ctx } from '@spraxium/components';
import type { ButtonInteraction } from 'discord.js';
import { ConfirmButton } from './confirm.button';

@ButtonHandler(ConfirmButton)
export class ConfirmButtonHandler {
  async handle(@Ctx() interaction: ButtonInteraction): Promise<void> {
    await interaction.reply({ content: 'Action confirmed.', ephemeral: true });
  }
}
````

</Tab>
</CodeTabs>

## Botões de link

Botões de link usam _@LinkButton_ e abrem uma URL no navegador do usuário ao serem clicados. Eles não têm _customId_ e não disparam handlers. A URL deve ser um endereço HTTP ou HTTPS válido. Botões de link são renderizados com o estilo _Link_ do Discord automaticamente.

```typescript filename="src/components/docs-link.button.ts"
import { LinkButton } from "@spraxium/components";

@LinkButton({
  url: "https://spraxium.dev/guide",
  label: "Documentation",
  emoji: "📚",
})
export class DocsLinkButton {}
```

## Renderização com ButtonService

O _ButtonService_ transforma metadados de decorator em action rows do discord.js. Dois métodos cobrem os diferentes cenários de build. Os action rows resultantes são passados para a propriedade _components_ da resposta de interação. O Discord limita em 5 botões por action row.

<Table>
  <TableHead>
    <TableRow>
      <TableCell header>Método</TableCell>
      <TableCell header>Assinatura</TableCell>
      <TableCell header>Finalidade</TableCell>
    </TableRow>
  </TableHead>
  <TableBody>
    <TableRow>
      <TableCell>*build*</TableCell>
      <TableCell>`build(...buttons: Type[])`</TableCell>
      <TableCell>Constrói uma action row a partir de uma ou mais classes de botão. Limitado a 5 botões.</TableCell>
    </TableRow>
    <TableRow>
      <TableCell>*rowWithContext*</TableCell>
      <TableCell>`rowWithContext(ctx: SpraxiumContext, ...buttons: Type[])`</TableCell>
      <TableCell>Constrói uma row que anexa o ID do contexto a cada *customId* de botão via separador til.</TableCell>
    </TableRow>

  </TableBody>
</Table>

<Callout type="tip">
  O Discord limita cada action row a 5 botões. O serviço lança um erro se você
  passar mais de cinco classes para *build*. Para layouts com mais botões, crie
  múltiplas rows chamando *build* separadamente e passe cada row no array
  *components*.
</Callout>

## Registro no módulo

Apenas os handlers precisam ser registrados, e eles vão no array _handlers_ do módulo de feature. Classes de componente decoradas com _@Button_ ou _@LinkButton_ são somente metadados e nunca são adicionadas a nenhum módulo. O dispatcher lê as configurações a partir da referência da classe armazenada dentro de cada decorator _@ButtonHandler_, sem precisar instanciar a classe do componente.

```typescript filename="src/components/components.module.ts"
import { Module } from "@spraxium/core";
import { ConfirmButtonHandler } from "./confirm.handler";
import { SelectItemHandler } from "./select-item.handler";

@Module({
  handlers: [ConfirmButtonHandler, SelectItemHandler],
})
export class ButtonsModule {}
```
