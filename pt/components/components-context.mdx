## O que é flow context

Flow context é um contêiner de dados de estado que encadeia informações entre interações sequenciais de componentes. Quando um usuário clica em um botão, seleciona uma opção e depois submete um modal, cada handler nessa sequência pode precisar de acesso a dados de etapas anteriores. Sem flow context, você precisaria codificar estado em strings de _customId_ (limitadas a 100 caracteres), armazenar dados em um banco externo ou usar mapas globais que vazam memória.

O _SpraxiumContext_ resolve isso fornecendo um contêiner de dados tipado, com limite de TTL e opcionalmente restrito a um usuário, que é armazenado em um adaptador plugável e referenciado por um UUID anexado ao _customId_ do componente. O _ContextService_ cria contextos, os serviços de componente anexam IDs de contexto a componentes renderizados, e o dispatcher resolve contextos automaticamente antes de invocar handlers.

O sistema de flow context é compartilhado entre todos os tipos de componente. Um contexto criado para uma row de botões pode ser lido por um handler de modal acionado posteriormente no mesmo fluxo. Esse encadeamento entre componentes é o que torna experiências interativas de múltiplas etapas possíveis sem gerenciamento manual de estado.

## Criando um contexto

Injete o _ContextService_ no seu comando ou serviço e chame _create_ com o objeto de dados que você quer encadear através do fluxo. O método retorna um _SpraxiumContext_ com um UUID gerado, os dados, timestamps e o TTL. Você então passa esse contexto para o método de build do serviço de componente.

```typescript filename="src/commands/setup-wizard.command-handler.ts"
import { Ctx, SlashCommandHandler } from '@spraxium/common';
import { ContextService, ButtonService } from '@spraxium/components';
import type { ChatInputCommandInteraction } from 'discord.js';
import { NextStepButton } from '../components/next-step.button';
import { SetupWizardCommand } from './setup-wizard.command';

@SlashCommandHandler(SetupWizardCommand)
export class SetupWizardCommandHandler {
  constructor(
    private readonly context: ContextService,
    private readonly buttons: ButtonService,
  ) {}

  async handle(@Ctx() interaction: ChatInputCommandInteraction): Promise<void> {
    const ctx = await this.context.create(
      { step: 1, guildId: interaction.guildId, settings: {} },
      { ttl: 600, restrictedTo: interaction.user.id },
    );

    const row = this.buttons.rowWithContext(ctx, NextStepButton);

    await interaction.reply({
      content: 'Welcome to the setup wizard. Click Next to begin.',
      components: [row],
    });
  }
}
```

## Interface SpraxiumContext

O objeto _SpraxiumContext_ carrega tanto seus dados customizados quanto metadados operacionais. O _id_ é um UUID que é anexado às strings de _customId_ via separador til. O campo _data_ contém seu payload tipado. Os campos _ttl_ e _expiresAt_ controlam a expiração automática, e _restrictedTo_ limita qual usuário pode interagir com componentes que carregam esse contexto.

## Lendo contexto nos handlers

Use o decorator de parâmetro _@FlowContext()_ no método _handle_ do seu handler para receber o contexto resolvido. O dispatcher extrai o ID do contexto do _customId_, recupera o contexto do adaptador de armazenamento, valida expiração e restrições de usuário, e injeta o objeto resolvido. Se o contexto tiver expirado ou o usuário estiver restrito, o dispatcher responde com a mensagem de erro configurada e não invoca o handler.

```typescript filename="src/components/next-step.handler.ts"
import { ButtonHandler, Ctx, FlowContext } from '@spraxium/components';
import type { ButtonInteraction } from 'discord.js';
import type { SpraxiumContext } from '@spraxium/components';
import { NextStepButton } from './next-step.button';

interface WizardData {
  step: number;
  guildId: string;
  settings: Record<string, unknown>;
}

@ButtonHandler(NextStepButton)
export class NextStepButtonHandler {
  async handle(
    @Ctx() interaction: ButtonInteraction,
    @FlowContext() ctx: SpraxiumContext<WizardData>,
  ): Promise<void> {
    const { step, settings } = ctx.data;
    await interaction.reply({
      content: `You are on step ${step}. Current settings: ${JSON.stringify(settings)}`,
      ephemeral: true,
    });
  }
}
```

## Atualizando contexto

O método _ContextService.update_ aplica um patch parcial nos dados do contexto sem substituir o objeto inteiro. Isso é útil em fluxos de múltiplas etapas onde cada handler adiciona ou modifica um subconjunto dos dados. O método retorna um booleano indicando se a atualização foi bem-sucedida (contexto encontrado e não expirado).

```typescript filename="src/components/step-two.handler.ts"
import { ButtonHandler, Ctx, FlowContext } from '@spraxium/components';
import { ContextService } from '@spraxium/components';
import type { ButtonInteraction } from 'discord.js';
import type { SpraxiumContext } from '@spraxium/components';
import { StepTwoButton } from './step-two.button';

interface WizardData {
  step: number;
  guildId: string;
  settings: Record<string, unknown>;
}

@ButtonHandler(StepTwoButton)
export class StepTwoButtonHandler {
  constructor(private readonly context: ContextService) {}

  async handle(
    @Ctx() interaction: ButtonInteraction,
    @FlowContext() ctx: SpraxiumContext<WizardData>,
  ): Promise<void> {
    await this.context.update<WizardData>(ctx.id, {
      step: 2,
      settings: { ...ctx.data.settings, language: 'en' },
    });

    await interaction.reply({
      content: 'Step 2 complete. Proceeding to step 3.',
      ephemeral: true,
    });
  }
}
```

## Excluindo contexto

Chame _ContextService.delete_ quando o fluxo estiver completo ou cancelado. Isso remove imediatamente o contexto do adaptador de armazenamento. Embora contextos expirem automaticamente com base no TTL, a exclusão explícita é uma boa prática para fluxos que completam normalmente, pois libera armazenamento e previne que dados obsoletos sejam lidos em casos extremos.

```typescript filename="src/components/finish-wizard.handler.ts"
import { ButtonHandler, Ctx, FlowContext } from '@spraxium/components';
import { ContextService } from '@spraxium/components';
import type { ButtonInteraction } from 'discord.js';
import type { SpraxiumContext } from '@spraxium/components';
import { FinishButton } from './finish.button';

@ButtonHandler(FinishButton)
export class FinishWizardHandler {
  constructor(private readonly context: ContextService) {}

  async handle(
    @Ctx() interaction: ButtonInteraction,
    @FlowContext() ctx: SpraxiumContext<unknown>,
  ): Promise<void> {
    await this.context.delete(ctx.id);
    await interaction.reply({ content: 'Setup complete!', ephemeral: true });
  }
}
```

## TTL e comportamento de expiração

Todo contexto tem um TTL medido em segundos. Quando um usuário interage com um componente cujo contexto expirou, o dispatcher responde com a mensagem configurada em _errorMessages.expired_ e não invoca o handler. Contextos expirados não são excluídos imediatamente do armazenamento; eles são limpos de forma lazy quando acessados ou durante varreduras periódicas do adaptador.

O TTL padrão é definido na configuração de _defineComponents_. Contextos individuais podem sobrescrevê-lo através do parâmetro _options.ttl_ em _ContextService.create_.

<Callout type="tip">
  Para interações curtas como diálogos de confirmação, um TTL de 60 a 120
  segundos é apropriado. Para wizards de múltiplas etapas, 600 a 1800 segundos
  oferece uma janela confortável para os usuários completarem o fluxo sem
  atingir a expiração no meio da interação.
</Callout>

## Restrições de usuário

A opção _restrictedTo_ limita o acesso ao contexto a um ID de usuário específico. Quando definida, apenas interações daquele ID de usuário são despachadas para o handler. Outros usuários que clicam no componente recebem a mensagem configurada em _errorMessages.restricted_. Isso é essencial para fluxos onde o componente é visível a todos no canal, mas deve ser acionável apenas pelo usuário que iniciou o comando.

```typescript filename="src/commands/verification.command-handler.ts"
import { Ctx, SlashCommandHandler } from '@spraxium/common';
import { ContextService, ButtonService } from '@spraxium/components';
import type { ChatInputCommandInteraction } from 'discord.js';
import { VerifyButton } from '../components/verify.button';
import { VerificationCommand } from './verification.command';

@SlashCommandHandler(VerificationCommand)
export class VerificationCommandHandler {
  constructor(
    private readonly context: ContextService,
    private readonly buttons: ButtonService,
  ) {}

  async handle(@Ctx() interaction: ChatInputCommandInteraction): Promise<void> {
    const ctx = await this.context.create(
      { userId: interaction.user.id },
      { ttl: 300, restrictedTo: interaction.user.id },
    );

    const row = this.buttons.rowWithContext(ctx, VerifyButton);

    await interaction.reply({
      content: 'Click the button to verify your identity.',
      components: [row],
    });
  }
}
```
