## Tipos de menu de seleção

O Discord fornece cinco variantes de menu de seleção, e o Spraxium suporta todas elas através de decorators dedicados. Selects de string permitem que você defina opções customizadas com labels e valores. Selects de user, role, mentionable e channel usam os seletores nativos de entidade do Discord que são preenchidos automaticamente com dados do servidor. Todos os cinco tipos compartilham o mesmo padrão de handler e sistema de injeção de parâmetros.

Cada tipo de select é definido como uma classe decorada, vinculado a um handler via um decorator de handler específico do tipo, e renderizado através do _SelectService_. A diferença principal entre os tipos é o decorator utilizado e o tipo de dado que o usuário seleciona. Selects de string requerem definições de opções explícitas, enquanto os outros quatro tipos preenchem suas opções a partir do servidor automaticamente.

## Selects de string com opções estáticas

Selects de string são o tipo mais flexível porque você controla as opções diretamente. Use o decorator empilhável _@SelectOption_ na mesma propriedade para definir cada opção. Cada opção tem um _label_ (exibido ao usuário), um _value_ (enviado na submissão) e uma _description_ opcional. A ordem dos decorators _@SelectOption_ determina a ordem das opções no menu renderizado.

<CodeTabs>
<Tab label="Componente">
```typescript filename="src/components/color-picker.select.ts"
import { StringSelect, SelectOption } from '@spraxium/components';

@StringSelect({
  customId: 'color-picker',
  placeholder: 'Choose a color',
  minValues: 1,
  maxValues: 1,
})
export class ColorPickerSelect {
@SelectOption({ label: 'Red', value: 'red', description: 'A warm color' })
@SelectOption({ label: 'Blue', value: 'blue', description: 'A cool color' })
@SelectOption({ label: 'Green', value: 'green', description: 'A natural color' })
options: void;
}

````
</Tab>
<Tab label="Handler">
```typescript filename="src/components/color-picker.handler.ts"
import { StringSelectHandler, Ctx, Selected } from '@spraxium/components';
import type { StringSelectMenuInteraction } from 'discord.js';
import { ColorPickerSelect } from './color-picker.select';

@StringSelectHandler(ColorPickerSelect)
export class ColorPickerHandler {
  async handle(
    @Ctx() interaction: StringSelectMenuInteraction,
    @Selected() color: string,
  ): Promise<void> {
    await interaction.reply({
      content: `You picked ${color}.`,
      ephemeral: true,
    });
  }
}
````

</Tab>
</CodeTabs>

## Referência de configuração

O objeto de configuração passado para _@StringSelect_ controla a aparência e o comportamento do menu. O _customId_ e o _placeholder_ são obrigatórios. Os campos _minValues_ e _maxValues_ controlam quantas opções o usuário pode selecionar simultaneamente.

## Opções dinâmicas

Quando as opções do select dependem de dados de runtime, como registros de banco de dados ou configurações do servidor, use a propriedade _dynamicOptions_ no decorator _@StringSelect_. Essa callback recebe um objeto de dados em tempo de build e retorna um array de objetos de opção.

<Callout type="tip">
  Os decorators *@SelectOption* e *dynamicOptions* são mutuamente exclusivos.
  Quando ambos estão presentes na mesma classe, a callback dinâmica tem
  precedência e as opções estáticas são completamente ignoradas.
</Callout>

```typescript filename="src/components/role-list.select.ts"
import { StringSelect } from '@spraxium/components';

@StringSelect({
  customId: 'role-list',
  placeholder: 'Pick a role',
  dynamicOptions: (roles: Array<{ id: string; name: string }>) =>
    roles.map((r) => ({
      label: r.name,
      value: r.id,
    })),
})
export class RoleListSelect {}
```

## Selects de entidade

Os selects de entidade usam os seletores nativos do Discord que exibem avatares de membros, cores de cargo e ícones de canal. Eles não precisam de decorators _@SelectOption_ porque as opções vêm do servidor automaticamente. Cada tipo de entidade tem seu próprio par de decorators para componente e handler.

## Injeção de parâmetros nos handlers

Handlers de select recebem os valores selecionados através de decorators de parâmetro. Use *@Selected()* para obter o primeiro valor selecionado como string, ou *@SelectedValues()* para obter todos os valores como um array de strings. O decorator *@Ctx()* fornece acesso à interação bruta do discord.js para cenários que precisam de mais controle.

```typescript filename="src/components/multi-role.handler.ts"
import { UserSelectHandler, Ctx, SelectedValues } from '@spraxium/components';
import type { UserSelectMenuInteraction } from 'discord.js';
import { AssignUserSelect } from './assign-user.select';

@UserSelectHandler(AssignUserSelect)
export class AssignUserHandler {
  async handle(
    @Ctx() interaction: UserSelectMenuInteraction,
    @SelectedValues() userIds: string[],
  ): Promise<void> {
    await interaction.reply({
      content: `Selected ${userIds.length} user(s).`,
      ephemeral: true,
    });
  }
}
````

## Renderização com SelectService

O _SelectService_ transforma metadados de decorator em action rows do discord.js. O método _build_ aceita a classe do select e dados opcionais (para _dynamicOptions_). O método _buildWithContext_ anexa um ID de flow context ao _customId_ do select.

```typescript filename="src/commands/pick-color.command-handler.ts"
import { Ctx, SlashCommandHandler } from '@spraxium/common';
import { SelectService } from '@spraxium/components';
import type { ChatInputCommandInteraction } from 'discord.js';
import { ColorPickerSelect } from '../components/color-picker.select';
import { PickColorCommand } from './pick-color.command';

@SlashCommandHandler(PickColorCommand)
export class PickColorCommandHandler {
  constructor(private readonly selects: SelectService) {}

  async handle(@Ctx() interaction: ChatInputCommandInteraction): Promise<void> {
    const row = await this.selects.build(ColorPickerSelect);

    await interaction.reply({
      content: 'Pick your favorite color:',
      components: [row],
    });
  }
}
```

## Construtor fluente alternativo

Para cenários onde você precisa construir definições de select programaticamente, use o construtor fluente _Selects.string()_. Esse construtor produz a mesma saída que os decorators, mas através de uma API de encadeamento de métodos. É útil em funções utilitárias ou quando a configuração do select depende inteiramente de dados de runtime.

```typescript filename="src/utils/select-factory.ts"
import { Selects } from '@spraxium/components';

export function buildCategorySelect(categories: string[]) {
  return Selects.string('category-select')
    .placeholder('Pick a category')
    .options(categories.map((c) => ({ label: c, value: c.toLowerCase() })))
    .build();
}
```

## Registro no módulo

Apenas os handlers precisam ser registrados, no array _handlers_ de um módulo de feature. Classes de componente de select decoradas com _@StringSelect_, _@UserSelect_ ou similares são somente metadados e nunca são adicionadas a nenhum módulo. O scanner de instâncias identifica selects pelo metadado do decorator no handler e os registra no _SelectDispatcher_ durante a fase de boot.

```typescript filename="src/components/components.module.ts"
import { Module } from '@spraxium/core';
import { ColorPickerHandler } from './color-picker.handler';
import { AssignUserHandler } from './assign-user.handler';

@Module({
  handlers: [ColorPickerHandler, AssignUserHandler],
})
export class SelectsModule {}
```
