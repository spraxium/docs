## O que é uma referência e por que ela existe

Uma referência é a ponte entre um diretório no disco e um namespace no espaço de chaves de tradução.
Você a declara dando a ela um nome e um caminho. O nome se torna o primeiro segmento de todas as
chaves que vêm daquele diretório. O caminho aponta para um diretório cuja estrutura segue uma de
duas convenções: arquivos JSON planos por locale, ou subdiretórios por locale contendo múltiplos
arquivos JSON.

A razão pela qual referências existem como conceito, em vez de um único diretório de locale plano, é que bots reais têm preocupações de tradução distintas que pertencem a lugares diferentes. Nomes e
descrições de slash commands são objetos de domínio na mesma categoria que o código. Mensagens de
erro usadas em todo o bot compartilham um namespace. Strings de interface para modals, menus de
seleção e embeds formam outra camada. Misturar tudo isso em um único arquivo por locale cria
overhead de manutenção e dificulta identificar o que está faltando onde.

## Declarando múltiplas referências

```typescript filename="config/i18n.config.ts"
import { defineI18n } from '@spraxium/i18n';

export const i18nConfig = defineI18n({
  defaultLocale: 'pt-BR',
  references: [
    { name: 'slash', path: './locales/slash' },
    { name: 'ui', path: './locales/ui' },
    { name: 'erros', path: './locales/erros' },
    { name: 'embeds', path: './locales/embeds' },
  ],
});
```

Não há limite de referências. Cada uma carrega de forma independente e é mesclada no mesmo registro.
Desde que cada valor de `name` seja único, não pode haver conflitos de chaves entre referências.

## Estrutura de arquivos planos

Nesse layout, cada locale é um único arquivo JSON na raiz do diretório da referência. O nome do
arquivo é o identificador do locale, e seu conteúdo é o objeto de tradução para aquele locale.

<FolderStructure>
  <Folder name="locales" open>
    <Folder name="erros" open>
      <File name="pt-BR.json" />
      <File name="en-US.json" />
      <File name="es-ES.json" />
    </Folder>
  </Folder>
</FolderStructure>

```json filename="locales/erros/pt-BR.json"
{
  'naoEncontrado': 'Recurso não encontrado',
  'proibido': 'Você não tem permissão para fazer isso',
  'timeout': 'A operação expirou após {{segundos}} segundos'
}
```

Após carregar a referência `{ name: 'erros', path: './locales/erros' }`, as chaves disponíveis no
registro para `pt-BR` são:

- `erros.naoEncontrado`
- `erros.proibido`
- `erros.timeout`

## Estrutura de diretórios aninhados

Quando uma referência contém muitos arquivos ou quando você quer organizar as traduções por funcionalidade
dentro de um locale, pode usar subdiretórios. Cada subdiretório tem o nome de um locale e contém
múltiplos arquivos JSON. O nome do arquivo (sem `.json`) se torna um segmento adicional de namespace.

<FolderStructure>
  <Folder name="locales" open>
    <Folder name="slash" open>
      <Folder name="pt-BR" open>
        <File name="ping.json" />
        <File name="ajuda.json" />
        <File name="config.json" />
      </Folder>
      <Folder name="en-US" open>
        <File name="ping.json" />
        <File name="ajuda.json" />
        <File name="config.json" />
      </Folder>
    </Folder>
  </Folder>
</FolderStructure>

```json filename="locales/slash/pt-BR/ping.json"
{
  'name': 'ping',
  'description': 'Verificar a latência do bot',
  'options': {
    'alvo': {
      'name': 'alvo',
      'description': 'O servidor ou serviço a verificar'
    }
  }
}
```

Após carregar a referência `{ name: 'slash', path: './locales/slash' }`, as chaves disponíveis para
`pt-BR` incluem `slash.ping.name`, `slash.ping.description`, `slash.ping.options.alvo.name`, e assim
por diante. O caminho completo é sempre: `nomeReferência.nomeArquivo.caminhoChaveJSON`.

## Acessando chaves entre referências

Todas as referências são acessíveis pela mesma API do serviço sem especificar de qual referência uma
chave pertence. O nome da referência é parte da própria chave.

```typescript filename="status.handler.ts"
import { Ctx, SlashCommandHandler } from '@spraxium/common';
import { I18nService } from '@spraxium/i18n';
import type { ChatInputCommandInteraction } from 'discord.js';
import { StatusCommand } from '../commands/status.command';

@SlashCommandHandler(StatusCommand)
export class StatusHandler {
  constructor(private readonly i18n: I18nService) {}

  async handle(@Ctx() interaction: ChatInputCommandInteraction): Promise<void> {
    const locale = interaction.locale;
    const userId = interaction.user.id;
    const guildId = interaction.guildId ?? undefined;

    // Chave da referência 'slash'
    const titulo = this.i18n.t('slash.status.embed.titulo', locale);

    // Chave da referência 'erros', usada quando uma verificação prévia falha
    const erroPerm = this.i18n.t('erros.proibido', locale);

    // Resolução assíncrona via cadeia de preferências do usuário/servidor
    const rodape = await this.i18n.tUser(
      userId,
      'slash.status.embed.rodape',
      {},
      guildId,
    );

    await interaction.reply({ content: `${titulo}: ${rodape}` });
  }
}
```

## Usando chaves de referência em decorators de slash

Quando você adiciona um campo `i18n` a um decorator de slash, os caminhos de chave apontam para o
mapa plano carregado, que usa o nome da referência como primeiro segmento.

```typescript filename="config.command.ts"
import { SlashCommand } from '@spraxium/common';

@SlashCommand({
  name: 'config',
  description: 'Gerenciar configurações do bot',
  i18n: {
    name: 'slash.config.name',
    description: 'slash.config.description',
  },
})
export class ConfigCommand {}
```

O lifecycle do i18n já carregou o mapa plano antes do registrador de slash ser executado. A bridge
consulta o mapa de cada locale suportado pelo bot e passa o resultado para
_builder.setNameLocalizations()_. O comando então aparece na interface do Discord com nomes
localizados nativamente.

## O que acontece quando um locale está faltando

Quando uma chave é solicitada para um locale sem arquivo carregado, o registro retorna silenciosamente
ao mapa do locale padrão. Se a chave também não existir no padrão, a string da chave bruta é retornada.
O sistema nunca lança erro em traduções ausentes durante as requisições. Degrada de forma segura em
produção, o que significa que você pode entregar traduções parciais: se você tem português e inglês e
quer adicionar espanhol futuramente, pode adicionar os arquivos `es-ES` incrementalmente.
