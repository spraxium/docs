## Como o array plugins funciona

Cada entrada no array _plugins_ é um objeto simples com duas propriedades: uma string _namespace_ que identifica o plugin de forma única e um payload _config_ com suas opções. Você nunca cria esse objeto manualmente. Os pacotes de plugin exportam helpers de factory tipados como _defineHttp_, _defineI18n_, _defineSchedule_ e _defineSignal_ que produzem entradas corretamente formatadas com inferência total de TypeScript. O framework resolve a configuração de cada plugin em runtime usando o namespace como chave.

```typescript filename="spraxium.config.ts"
import { defineConfig } from '@spraxium/core';
import { defineHttp } from '@spraxium/http';
import { defineI18n } from '@spraxium/i18n';
import { defineSchedule } from '@spraxium/schedule';

export default defineConfig({
  plugins: [
    defineI18n({
      defaultLocale: 'en-US',
      references: [{ name: 'commands', path: './locales' }],
    }),
    defineHttp({
      enabled: true,
      port: 3001,
      host: 'localhost',
      apiKey: process.env.HTTP_API_KEY ?? 'dev-secret',
      sharding: false,
    }),
    defineSchedule({
      timezone: 'America/Sao_Paulo',
      defaultLockTtlMs: 60_000,
      disableAll: false,
    }),
  ],
});
```

## Extraindo a configuração de plugins para arquivos separados

Em projetos com vários plugins, manter todas as chamadas de factory inline no `spraxium.config.ts` dificulta a revisão do arquivo. O padrão recomendado é extrair a chamada de factory de cada plugin para um arquivo dedicado dentro de `config/`, exportar o resultado como uma constante nomeada e importá-la na configuração raiz. Isso mantém as opções de cada plugin isoladas e auditáveis de forma independente.

<CodeTabs>
  <Tab label="config/schedule.config.ts" language="typescript">
```typescript filename="config/schedule.config.ts"
import { defineSchedule } from '@spraxium/schedule';

export const schedulePlugin = defineSchedule({
  timezone: 'America/Sao_Paulo',
  defaultLockTtlMs: 60_000,
  disableAll: false,
});
```
  </Tab>
  <Tab label="config/http.config.ts" language="typescript">
```typescript filename="config/http.config.ts"
import { defineHttp } from '@spraxium/http';

export const httpPlugin = defineHttp({
  enabled: true,
  port: 3001,
  host: 'localhost',
  apiKey: process.env.HTTP_API_KEY ?? 'dev-secret',
  sharding: false,
});
```
  </Tab>
  <Tab label="spraxium.config.ts" language="typescript">
```typescript filename="spraxium.config.ts"
import { defineConfig } from '@spraxium/core';
import { httpPlugin } from './config/http.config';
import { schedulePlugin } from './config/schedule.config';

export default defineConfig({
  plugins: [httpPlugin, schedulePlugin],
});
```
  </Tab>
</CodeTabs>

<Callout
  tone="success"
  title="Mantenha a configuração próxima do import da factory"
>
  Quando o namespace e o payload ficam visíveis juntos no mesmo arquivo,
  auditorias e refatorações são bem mais rápidas. Evite dispersar opções de
  plugin em módulos utilitários ou arquivos de constantes compartilhadas que
  também são usados para outros fins.
</Callout>
