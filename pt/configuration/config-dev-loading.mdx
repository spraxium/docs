## O bloco dev

O bloco _dev_ controla o watcher de desenvolvimento local que reinicia o processo do seu bot quando arquivos mudam. Todos os campos são opcionais. O entrypoint padrão é `src/main.ts` e o debounce padrão é 300 milissegundos. Na maioria dos projetos, você só precisa ajustar o _debounce_ para evitar storms de reinício durante salvamentos rápidos.

```typescript filename="spraxium.config.ts"
import { defineConfig } from "@spraxium/core";

export default defineConfig({
  dev: {
    entrypoint: "src/main.ts",
    debounce: 500,
    include: ["locales"],
    exclude: [".spraxium/**", "dist/**"],
  },
});
```

<Callout tone="warning" title="Exclua diretórios gerados">
  Sempre adicione .spraxium/** e dist/** à lista *exclude*. Esses diretórios
  mudam durante operações de build e, se observados, causam loops infinitos de
  reinício que desestabilizam o processo do watcher.
</Callout>

## Resolução de ambiente

O `NODE_ENV` é lido uma vez durante a inicialização e mapeado para um objeto _ConfigEnv_. Esse valor nunca muda após o carregamento da configuração, portanto todo branching baseado em ambiente deve acontecer dentro do callback do _defineConfig_.
