## debug

A flag _debug_ habilita saída diagnóstica extra em todos os subsistemas. Quando ativa, o framework emite linhas de rastreamento adicionais durante o carregamento de módulos, resolução de providers, registro de comandos e despacho de interações. Essa saída é destinada a troubleshooting e não deve permanecer ativa em deployments de produção com alto volume de tráfego.

A flag aceita um booleano simples ou qualquer expressão de _ConfigEnv_, permitindo escopar a verbosidade a um ambiente específico.

## logger

O bloco _logger_ controla a política de saída do logger integrado do framework. Não afeta lógica de negócio. As opções mais configuradas são _maskTokens_ (substitui strings parecidas com tokens por um placeholder seguro), _commandLogging_ (emite uma linha de rastreamento a cada despacho de comando) e _discord_ (envia níveis de log selecionados para um canal via webhook).

```typescript filename="config/logger.config.ts"
import type { SpraxiumLoggerConfig } from '@spraxium/core';

export const loggerConfig: SpraxiumLoggerConfig = {
  maskTokens: true,
  commandLogging: true,
  discord: {
    type: 'channel',
    webhookUrl: process.env.LOGGER_WEBHOOK!,
    levels: ['error', 'warn'],
    channelId: '1462519588775395429',
  },
};
```

## plugins

O array _plugins_ coleta entradas retornadas pelas factories de plugin. Cada entrada carrega uma string _namespace_ e um payload de configuração tipado. O framework usa o namespace para rotear a configuração para o plugin correto em runtime. As factories de plugin e as opções first-party são abordadas na página de Registro de Plugins.

## dev

O bloco _dev_ tem uma página de referência dedicada nesta categoria, cobrindo todos os campos do watcher, os nomes de arquivo de configuração suportados e como os ambientes são resolvidos na inicialização.

O bloco _prefix_ é documentado na categoria de Comandos de Prefixo. O bloco _exceptions_ é documentado na categoria de Exceções.
