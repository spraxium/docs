## A classe de definição de comando

Uma definição de slash command é uma classe decorada com _@SlashCommand_ que declara o que o Discord deve exibir no seletor de comandos. A classe em si não contém lógica de execução. Seu propósito é manter metadados que o _SlashRegistrar_ lê na inicialização para construir o payload JSON enviado à API REST do Discord. Opções são declaradas como decorators de método em um método _build()_, e subcomandos são declarados em métodos nomeados. A classe pode ser importada por handlers, handlers de autocomplete e fixtures de teste sem trazer dependências de services.

O decorator _@SlashCommand_ aceita um objeto _SlashCommandConfig_ com as seguintes propriedades. Todo comando deve fornecer _name_ e _description_ no mínimo. As propriedades restantes controlam permissões padrão, disponibilidade em DM, flag NSFW, registro específico de guild e localização.

<DropdownGroup>
  <Dropdown title="name, string (obrigatório)">
    O nome do comando exibido no seletor do Discord. Deve ser minúsculo, entre 1
    e 32 caracteres, sem espaços. É o identificador que o Discord usa para
    rotear a interação ao seu bot. Alterar o nome do comando após o registro
    cria uma nova entrada no registry do Discord; o nome antigo não é removido
    automaticamente até que você force um novo registro.
  </Dropdown>
  <Dropdown title="description, string (obrigatório)">
    A descrição exibida abaixo do nome do comando no seletor do Discord. Aceita
    de 1 a 100 caracteres. Escreva uma descrição concisa e orientada à ação para
    que os usuários entendam o propósito do comando antes de invocá-lo.
    Descrições são indexadas pelo Discord para busca, então textos claros
    melhoram a descoberta do comando.
  </Dropdown>
  <Dropdown title="defaultMemberPermissions, bigint | number | null">
    Bitfield de permissões necessárias para ver e usar o comando. Use
    `PermissionFlagsBits` do discord.js para compor o valor. Quando definido, o
    Discord oculta o comando de membros que não possuem as permissões
    especificadas. Defina como `null` para tornar o comando visível a todos por
    padrão. Administradores de servidor podem sobrescrever esse valor por guild
    nas configurações do servidor.
  </Dropdown>
  <Dropdown title="dmPermission, boolean">
    Controla se o comando está disponível em mensagens diretas. Padrão `true`
    quando omitido. Defina como `false` para comandos que exigem contexto de
    guild, como os que leem membros ou canais. Essa flag não tem efeito em
    comandos exclusivos de guild criados via propriedade *guild*.
  </Dropdown>
  <Dropdown title="nsfw, boolean">
    Marca o comando como restrito por idade. Quando `true`, o comando aparece
    apenas em canais NSFW e em DMs para usuários que confirmaram ter idade
    adequada. Use essa flag para comandos que apresentam conteúdo adulto, em
    conformidade com as políticas do Discord.
  </Dropdown>
  <Dropdown title="guild, string">
    ID da guild para registro específico de guild. Quando fornecido, o comando é
    registrado apenas naquela guild e ignora o registro global. Comandos de
    guild propagam em segundos, em vez do atraso de até uma hora dos comandos
    globais. Remova a propriedade *guild* antes de implantar em produção para
    registrar o comando globalmente.
  </Dropdown>
  <Dropdown title="i18n, SlashI18nKeys">
    Chaves de localização para nome e descrição do comando, resolvidas pelo
    módulo i18n. Fornecer um objeto *SlashI18nKeys* com as chaves *name* e
    *description* que mapeiam para entradas nos seus arquivos de locale. Os
    campos *name* e *description* base servem como fallback quando nenhum locale
    correspondente for encontrado para o idioma do cliente Discord do usuário.
  </Dropdown>
</DropdownGroup>

```typescript filename="commands/ban.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';
import { PermissionFlagsBits } from 'discord.js';

@SlashCommand({
  name: 'ban',
  description: 'Ban a member from the server.',
  defaultMemberPermissions: PermissionFlagsBits.BanMembers,
})
export class BanCommand {
  @SlashOption.User('target', {
    description: 'The member to ban',
    required: true,
  })
  @SlashOption.String('reason', {
    description: 'Reason for the ban',
    required: false,
    maxLength: 512,
  })
  @SlashOption.Integer('days', {
    description: 'Number of days of messages to delete',
    required: false,
    min: 0,
    max: 7,
  })
  build() {}
}
```

## Tipos de opções

O Spraxium fornece nove decorators de tipo de opção através do namespace _SlashOption_, um para cada tipo de opção de comando de aplicação Discord. Cada decorator é um decorator de método aplicado ao método _build()_ da classe de comando (ou a um método _@SlashSubcommand_ quando a opção pertence a um subcomando). Opções são empilhadas no mesmo método, e a ordem dos decorators determina a ordem de exibição no client Discord (decorator do topo aparece por último no seletor).

<DropdownGroup>
  <Dropdown title="@SlashOption.String, string | null">
    Mapeia para o tipo STRING do Discord. O handler recebe `string` quando o
    usuário preenche a opção, ou `null` quando a opção não é obrigatória e foi
    omitida. Suporta *minLength*, *maxLength*, *choices* (lista estática) e
    *autocomplete* (sugestões dinâmicas). *choices* e *autocomplete* não podem
    ser combinados na mesma opção.
  </Dropdown>
  <Dropdown title="@SlashOption.Integer, number | null">
    Mapeia para o tipo INTEGER do Discord. Aceita apenas números inteiros. O
    handler recebe `number` ou `null`. Suporta *min* e *max* para validação de
    intervalo, *choices* para um conjunto fixo de inteiros e *autocomplete* para
    sugestões dinâmicas.
  </Dropdown>
  <Dropdown title="@SlashOption.Number, number | null">
    Mapeia para o tipo NUMBER do Discord. Aceita números de ponto flutuante. O
    handler recebe `number` ou `null`. Suporta *min* e *max* para validação de
    intervalo e *autocomplete* para sugestões dinâmicas. *choices* estáticos não
    estão disponíveis para o tipo Number.
  </Dropdown>
  <Dropdown title="@SlashOption.Boolean, boolean | null">
    Mapeia para o tipo BOOLEAN do Discord. O Discord renderiza como um toggle
    verdadeiro/falso no seletor de comandos. O handler recebe `true`, `false` ou
    `null` quando a opção é opcional e não foi preenchida. Nenhum constraint
    adicional está disponível para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.User, User | null">
    Mapeia para o tipo USER do Discord. O Discord renderiza um seletor de
    usuário. O handler recebe um objeto *User* completamente resolvido do
    discord.js, ou `null` se a opção for opcional e não foi preenchida. Nenhum
    constraint adicional está disponível para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.Channel, Channel | null">
    Mapeia para o tipo CHANNEL do Discord. O Discord renderiza um seletor de
    canal. O handler recebe um objeto *Channel* ou `null`. Suporta o array
    *channelTypes* para restringir quais tipos de canal o usuário pode
    selecionar. Veja a seção de filtragem de tipos de canal abaixo para a lista
    completa de valores aceitos.
  </Dropdown>
  <Dropdown title="@SlashOption.Role, Role | null">
    Mapeia para o tipo ROLE do Discord. O Discord renderiza um seletor de cargo.
    O handler recebe um objeto *Role* completamente resolvido do discord.js, ou
    `null` se a opção for opcional e não foi preenchida. Nenhum constraint
    adicional está disponível para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.Mentionable, User | Role | null">
    Mapeia para o tipo MENTIONABLE do Discord. Aceita tanto usuários quanto
    cargos. O handler recebe um objeto *User* ou *Role* dependendo do que foi
    selecionado, ou `null` se opcional e omitido. Nenhum constraint adicional
    está disponível para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.Attachment, Attachment | null">
    Mapeia para o tipo ATTACHMENT do Discord. O Discord renderiza um campo de
    upload de arquivo. O handler recebe um objeto *Attachment* do discord.js com
    metadados e URL CDN, ou `null` se a opção for opcional e não foi preenchida.
    Nenhum constraint adicional está disponível para este tipo.
  </Dropdown>
</DropdownGroup>

Cada decorator de opção aceita dois argumentos posicionais: o _name_ da opção (string, usado para registro no Discord e correspondência com _@SlashOpt_) e um objeto _config_. O objeto config estende _SlashBaseOptionConfig_ com campos específicos do tipo.

## Constraints de opções string

O decorator _@SlashOption.String_ suporta quatro propriedades de configuração adicionais além da config base. Essas propriedades mapeiam diretamente para os constraints da API Discord para opções string e permitem controlar o comprimento do input, fornecer choices fixos ou habilitar sugestões de autocomplete.

```typescript filename="commands/echo.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';

@SlashCommand({
  name: 'echo',
  description: 'Repeat a message back to the channel.',
})
export class EchoCommand {
  @SlashOption.String('message', {
    description: 'Text to echo',
    required: true,
    minLength: 1,
    maxLength: 2000,
  })
  build() {}
}
```

Quando _minLength_ ou _maxLength_ é definido, o Discord valida o comprimento do input no lado do client antes da interação ser enviada. Quando _choices_ é fornecido, o usuário seleciona de um dropdown em vez de digitar livremente. Quando _autocomplete_ é definido como `true`, o Discord chama seu handler de autocomplete conforme o usuário digita, fornecendo sugestões dinâmicas em tempo real.

<Callout
  tone="warning"
  title="choices e autocomplete são mutuamente exclusivos"
>
  Definir *autocomplete* como true e fornecer um array *choices* na mesma opção
  não é permitido pela API do Discord. Quando *autocomplete* está ativado, os
  choices estáticos são ignorados e apenas seu handler de autocomplete é
  consultado. Use *choices* estáticos para listas pequenas e fixas e
  *autocomplete* para sugestões dinâmicas ou vindas de banco de dados que
  excedam 25 entradas.
</Callout>

## Constraints de opções numéricas

Tanto _@SlashOption.Integer_ quanto _@SlashOption.Number_ suportam propriedades _min_ e _max_ que definem o intervalo de valores permitidos. O Discord impõe esses constraints no lado do client, então o handler tem garantia de receber um valor dentro do intervalo especificado (ou `null` se a opção não for obrigatória e o usuário a omitir). A variante _Integer_ também suporta _choices_ e _autocomplete_, enquanto a variante _Number_ suporta apenas _autocomplete_.

## Filtragem de tipo de canal

O decorator _@SlashOption.Channel_ aceita um array _channelTypes_ que restringe quais tipos de canal o usuário pode selecionar. Os valores correspondem a membros do enum _ChannelType_ do Discord. Quando o array é fornecido, o client Discord mostra apenas canais correspondentes aos tipos especificados no seletor de opções.

```typescript filename="commands/setup.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';
import { ChannelType } from 'discord.js';

@SlashCommand({
  name: 'setup',
  description: 'Configure the bot for this server.',
})
export class SetupCommand {
  @SlashOption.Channel('log-channel', {
    description: 'Channel for bot logs',
    required: true,
    channelTypes: [ChannelType.GuildText, ChannelType.GuildAnnouncement],
  })
  @SlashOption.Channel('voice-channel', {
    description: 'Default voice channel',
    required: false,
    channelTypes: [ChannelType.GuildVoice, ChannelType.GuildStageVoice],
  })
  build() {}
}
```

## Localização com chaves i18n

Tanto o decorator _@SlashCommand_ quanto cada decorator _@SlashOption_ aceitam uma propriedade opcional _i18n_ do tipo _SlashI18nKeys_. Esse objeto contém duas chaves string, _name_ e _description_, que o módulo i18n resolve para strings específicas de locale durante o registro do comando. As localizações resolvidas são enviadas ao Discord como parte do payload do comando, para que os usuários vejam nomes de comandos e opções no idioma do seu client Discord.

```typescript filename="commands/locale.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';

@SlashCommand({
  name: 'locale',
  description: 'Change your preferred language.',
  i18n: {
    name: 'commands.locale.name',
    description: 'commands.locale.description',
  },
})
export class LocaleCommand {
  @SlashOption.String('language', {
    description: 'The language to use.',
    required: true,
    i18n: {
      name: 'commands.locale.options.language.name',
      description: 'commands.locale.options.language.description',
    },
    choices: [
      { name: 'English (US)', value: 'en-US' },
      { name: 'Português (Brasil)', value: 'pt-BR' },
    ],
  })
  build() {}
}
```

As chaves _i18n_ seguem o mesmo padrão de resolução do restante do sistema i18n do framework. Elas mapeiam para chaves nos seus arquivos JSON de locale e suportam todos os locales suportados pelo Discord. Quando um usuário com client Discord em Português Brasileiro abre o seletor de comandos, ele vê os nomes traduzidos do comando e das opções automaticamente.

## Checklist de design para definições de comandos

<Steps>
  <Step title="Mantenha definições livres de lógica de negócio">
    A classe de comando deve conter apenas decorators de metadados e um método
    *build()* vazio (ou métodos nomeados de subcomando). Nunca importe services,
    repositórios ou dependências de runtime em um arquivo de definição de
    comando. Isso garante que a classe possa ser importada por handlers,
    handlers de autocomplete e testes sem criar cadeias de dependência.
  </Step>
  <Step title="Use o tipo de opção mais específico disponível">
    Prefira *@SlashOption.User* sobre *@SlashOption.String* para IDs de usuário,
    *@SlashOption.Channel* sobre *@SlashOption.String* para referências de
    canal, e *@SlashOption.Integer* sobre *@SlashOption.Number* para números
    inteiros. Tipos específicos dão ao Discord melhores affordances de UI como
    seletores de usuário e dropdowns de canal, e o handler recebe objetos
    discord.js completamente resolvidos em vez de strings brutas.
  </Step>
  <Step title="Restrinja canais com channelTypes">
    Quando um comando espera um tipo específico de canal, sempre forneça o array
    *channelTypes*. Isso impede que usuários selecionem canais de voz onde você
    espera canais de texto, e o client Discord impõe o constraint antes da
    interação chegar ao seu bot.
  </Step>
  <Step title="Prefira choices sobre autocomplete para conjuntos pequenos">
    Arrays estáticos de *choices* são exibidos como dropdown e não requerem
    código de handler. Use-os quando o conjunto de valores válidos é pequeno e
    fixo (abaixo de 25 entradas). Use *autocomplete* quando os valores são
    dinâmicos, vêm de um banco de dados ou excedem o limite de 25 choices.
  </Step>
  <Step title="Defina permissões no nível da definição">
    Use *defaultMemberPermissions* para definir as permissões mínimas
    necessárias para ver o comando no seletor. Esta é uma dica de UI que o
    Discord impõe no lado do client. Combine com *@UseGuards* no handler para
    enforcement do lado do servidor que não pode ser contornado.
  </Step>
</Steps>
