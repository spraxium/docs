## Visao geral

O repositorio do Spraxium inclui varios apps em `apps/`. Cada um foi criado para demonstrar um conjunto especifico de recursos em um projeto pequeno, legivel e executavel localmente.

A documentacao explica o conceito. Os examples mostram como isso fica em codigo real.

Repositorio: [github.com/spraxium/spraxium/tree/main/apps](https://github.com/spraxium/spraxium/tree/main/apps)

## Mapa dos examples

### slash-bot

**Serve para:** slash commands, subcommands, guards e autocomplete.

**Como usar:** comece por ele se seu bot for command-driven. Abra `simple`, depois `subcommands`, depois `protection` para entender o pipeline completo.

Pontos principais:

- Handlers com `@SlashCommandHandler`, `@Ctx`, `@SlashOpt`.
- Arvore de subcomandos (`/math`, `/server`).
- Guardas com `@UseGuards(GuildOnly, PermissionGuard)`.
- Autocomplete com `@OnAutocomplete`.

### modal-bot

**Serve para:** formularios com modal e processamento de submit.

**Como usar:** estude os modulos `profile`, `suggestion` e `ticket` para ver padroes de abertura e tratamento de modal.

Pontos principais:

- Definicao de campos no componente modal.
- Handler de submit separado do comando que abre modal.
- Fluxo de abrir e editar no modulo de ticket.

### components-bot

**Serve para:** botoes, selects, modais, flow context e layouts V2.

**Como usar:** use como referencia geral de `@spraxium/components`. E o app mais completo para interface interativa.

Pontos principais:

- `buttons/`: estilos e acoes de botao.
- `selects/`: string, user, role, channel e multi-select.
- `flow/`: wizard com estado por sessao usando FlowContext.
- `v2/`: exemplos class-based, fluent e dinamico para containers V2.

### http-bot

**Serve para:** expor estado do bot via API HTTP.

**Como usar:** veja `status.controller.ts` + `status-http.service.ts` e depois `config/http.config.ts` para entender o plugin.

Pontos principais:

- `@HttpController`, `@HttpGet`, `@HttpCtx`.
- Service injetado via DI no controller.
- Plugin com `defineHttp({ port, apiKey, module })`.

### i18n-bot

**Serve para:** internacionalizacao por usuario e comandos de locale.

**Como usar:** leia `greet` para traducao por usuario (`tUser`) e `locale` para alterar e consultar preferencia.

Pontos principais:

- Locale por usuario com fallback.
- Arquivos `locales/en-US.json` e `locales/pt-BR.json`.
- Interpolacao de variaveis nas mensagens.

### schedule-bot

**Serve para:** jobs agendados com `@spraxium/schedule`.

**Como usar:** veja `startup-tasks.service.ts` para jobs de inicializacao e `scheduled-tasks.service.ts` para jobs recorrentes.

Pontos principais:

- `@AfterOnline` e `@Timeout` para eventos one-shot.
- `@Interval` e `@Cron` para execucao recorrente.
- Config do plugin em `config/schedule.config.ts`.

### signal-bot

**Serve para:** receber sinais assincronos por webhook do Discord.

**Como usar:** comece por `config/signal.config.ts`, depois veja `signals/` e `schemas/` para entender roteamento e validacao.

Pontos principais:

- `@SignalListener` + `@OnSignal(event)`.
- Validacao com Zod em `schemas/`.
- Separacao entre schema e logica do handler.

### sandbox

**Serve para:** app amplo com multiplos modulos em um unico projeto.

**Como usar:** use como referencia quando voce precisar combinar varios pacotes no mesmo bot (components, schedule, i18n, http, exceptions, etc).

Pontos principais:

- Estrutura de projeto mais proxima de producao.
- Muitos modulos coexistindo no mesmo `AppModule`.
- Bom para estudar organizacao em escala.

## Como rodar qualquer example

No root do repositorio:

```bash
pnpm install
```

Depois, por exemplo com `slash-bot`:

```bash
cp apps/slash-bot/.env.example apps/slash-bot/.env
# preencha DISCORD_TOKEN no .env
cd apps/slash-bot
pnpm dev
```

> Os examples sao pensados para estudo e teste local. Use como base e adapte para seu ambiente real (seguranca, observabilidade, persistencia e deploy).
