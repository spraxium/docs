## O que o módulo de ambiente faz

O pacote *@spraxium/env* substitui o padrão comum de ler *process.env* diretamente e torcer para que os valores estejam presentes em runtime. Em vez de espalhar acessos ao environment por toda a base de código, você define uma única classe tipada que declara exatamente quais variáveis o bot precisa, qual o tipo de cada uma e quais são opcionais. Quando o bot inicia, o `EnvValidator.validate` lê o arquivo *.env*, verifica cada campo contra essas regras e retorna um objeto completamente tipado ou lança um erro claro listando exatamente o que está errado.

O benefício prático é que falhas de configuração aparecem durante o startup, não após o deploy quando um usuário acionar a feature afetada. Uma URL de banco de dados ausente ou um endpoint de webhook malformado se torna um erro de startup com mensagem descritiva em vez de um crash enigmático em runtime. Esse comportamento fail-fast é especialmente valioso em ambientes de produção onde debugar problemas ao vivo custa significativamente mais tempo do que capturá-los durante a sequência de boot.

## Definir o schema

Crie uma classe em *src/app.env.ts* que estenda `SpraxiumBaseEnv` para herdar os campos base que todo bot Spraxium precisa. A classe base já inclui *TOKEN* e *NODE_ENV*, então você só precisa adicionar os campos específicos da sua aplicação. Cada campo usa o decorator `@Env()` para vinculá-lo a uma variável de ambiente nomeada, e você pode empilhar decorators de validação como `@IsString()` e `@IsUrl()` para impor regras de formato automaticamente.

```typescript filename="src/app.env.ts"
import { Env, EnvSchema, IsOptional, IsString, IsUrl, SpraxiumBaseEnv } from '@spraxium/env';

@EnvSchema()
export class AppEnv extends SpraxiumBaseEnv {
  @Env('DATABASE_URL')
  @IsString()
  DATABASE_URL!: string;

  @Env('DISCORD_WEB', { secret: false })
  @IsString()
  @IsUrl()
  DISCORD_WEB!: string;

  @Env('ANALYTICS_TRACKING_ID', { secret: false })
  @IsString()
  @IsOptional()
  ANALYTICS_TRACKING_ID?: string;
}
```

A flag *secret* controla se o valor aparece em logs e saídas de diagnóstico, o que é importante para manter credenciais sensíveis como tokens e chaves de API fora dos streams de log em produção. Campos sem `@IsOptional()` são obrigatórios por padrão e falharão na validação se estiverem ausentes do ambiente, garantindo que o bot nunca inicie com configuração incompleta.

## Validar antes do boot

Chame `EnvValidator.validate` como a primeira coisa em *src/main.ts*, antes de criar a instância da aplicação ou carregar qualquer módulo. Isso garante que toda variável de ambiente foi verificada e tipada antes de qualquer código do framework rodar, e lhe dá um objeto completamente tipado que pode ser passado via injeção de dependência ou usado diretamente na lógica de bootstrap.

```typescript filename="src/main.ts"
import { EnvValidator } from '@spraxium/env';
import { AppEnv } from './app.env';

const env = EnvValidator.validate(AppEnv);
```

Após essa chamada, a variável *env* é uma instância completamente tipada de *AppEnv* com todas as validações aprovadas. Você pode passar *env.token* para `SpraxiumFactory.create`, usar *env.DATABASE_URL* para inicializar uma conexão com banco de dados, ou injetar o objeto inteiro via DI em qualquer provider que precise de valores de configuração. Se qualquer campo obrigatório estiver ausente ou falhar nas regras de validação, o processo encerra imediatamente com uma mensagem clara antes de qualquer módulo ou provider ser instanciado.
