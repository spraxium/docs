## Visão geral dos tipos de campo

Modais do Spraxium suportam 11 tipos de campo, cada um declarado através de um decorator no nível da propriedade na sua classe _@ModalComponent_. Inputs de texto e selects são os mais comuns, mas o sistema também inclui grupos de rádio, checkboxes, uploads de arquivo e exibições de texto somente leitura. Cada decorator de campo aceita um objeto de configuração com propriedades específicas do tipo, além de propriedades compartilhadas como _label_ e _required_.

A ordem dos campos no modal renderizado corresponde à ordem de declaração das propriedades decoradas na classe. Modais do Discord têm um máximo de 5 action rows, e cada campo ocupa uma row. Esse limite significa que um único modal pode conter no máximo 5 campos, embora campos dinâmicos e condicionais possam alterar o que aparece para cada usuário.

## Inputs de texto

O decorator _@ModalInput_ cria um campo de input de texto. A propriedade _style_ controla se o campo renderiza como uma entrada de linha única (_short_) ou multi-linha (_paragraph_). Campos curtos são ideais para nomes, assuntos e valores breves. Campos de parágrafo são adequados para descrições, mensagens e conteúdo mais longo.

```typescript filename="src/components/report.modal.ts"
import { ModalComponent, ModalInput } from '@spraxium/components';

@ModalComponent({ id: 'report-modal', title: 'Submit Report' })
export class ReportModal {
  @ModalInput({
    label: 'Title',
    placeholder: 'A short title for the report',
    required: true,
    minLength: 5,
    maxLength: 100,
    style: 'short',
  })
  title!: string;

  @ModalInput({
    label: 'Description',
    placeholder: 'Describe the issue in detail...',
    required: true,
    minLength: 20,
    maxLength: 4000,
    style: 'paragraph',
  })
  description!: string;

  @ModalInput({
    label: 'Reproduction Steps',
    placeholder: 'Optional steps to reproduce',
    required: false,
    style: 'paragraph',
    value: "1. \n2. \n3. ",
  })
  steps!: string;
}
```

## Selects de string com choices

O decorator _@ModalSelect_ cria um menu de seleção dropdown dentro do modal. Defina as opções disponíveis usando o decorator empilhável _@ModalChoice_ na mesma propriedade. Cada choice tem um _label_ (exibido ao usuário), _value_ (enviado na submissão) e uma _description_ opcional.

```typescript filename="src/components/category-select.modal.ts"
import { ModalComponent, ModalSelect, ModalChoice } from '@spraxium/components';

@ModalComponent({ id: 'ticket-modal', title: 'Create Ticket' })
export class TicketModal {
  @ModalSelect({
    label: 'Category',
    placeholder: 'Select a ticket category',
    required: true,
  })
  @ModalChoice({
    label: 'Bug Report',
    value: 'bug',
    description: 'Something is broken',
  })
  @ModalChoice({
    label: 'Feature Request',
    value: 'feature',
    description: 'Suggest an improvement',
  })
  @ModalChoice({
    label: 'General Question',
    value: 'question',
    description: 'Ask a question',
  })
  category!: string;

  @ModalInput({
    label: 'Details',
    style: 'paragraph',
    required: true,
  })
  details!: string;
}
```

## Selects de entidade

Decorators de select de entidade permitem que os usuários selecionem membros do servidor, cargos, canais ou entidades mencionáveis diretamente no modal. Esses campos usam os seletores nativos de entidade do Discord, exibindo avatares, cores de cargo e ícones de canal. Nenhum decorator _@ModalChoice_ é necessário porque as opções são preenchidas a partir do servidor.

<CodeTabs>
<Tab label="Select de Usuário">
```typescript filename="src/components/assign.modal.ts"
import { ModalComponent, ModalUserSelect } from '@spraxium/components';

@ModalComponent({ id: 'assign-modal', title: 'Assign User' })
export class AssignModal {
  @ModalUserSelect({
    label: 'Assignee',
    placeholder: 'Pick a team member',
    required: true,
  })
  assignee!: string;
}

````
</Tab>
<Tab label="Select de Canal">
```typescript filename="src/components/move-channel.modal.ts"
import { ModalComponent, ModalChannelSelect } from '@spraxium/components';
import { ChannelType } from 'discord.js';

@ModalComponent({ id: 'move-modal', title: 'Move to Channel' })
export class MoveChannelModal {
  @ModalChannelSelect({
    label: 'Target Channel',
    placeholder: 'Select a text channel',
    required: true,
    channelTypes: [ChannelType.GuildText],

}
````

</Tab>

## Grupos de rádio e checkboxes

Grupos de rádio apresentam opções mutuamente exclusivas onde o usuário deve escolher exatamente uma. Grupos de checkbox permitem seleções múltiplas. Um *@ModalCheckbox* único renderiza como um toggle booleano. Todos os campos baseados em choices usam *@ModalOption* (um alias para *@ModalChoice*) para definir suas opções.

<CodeTabs>
<Tab label="Grupo de Rádio">
```typescript filename="src/components/priority.modal.ts"
import { ModalComponent, ModalRadioGroup, ModalOption } from '@spraxium/components';

@ModalComponent({ id: 'priority-modal', title: 'Set Priority' })
export class PriorityModal {
  @ModalRadioGroup({ label: 'Priority Level' })
  @ModalOption({ label: 'Low', value: 'low' })
  @ModalOption({ label: 'Medium', value: 'medium' })
  @ModalOption({ label: 'High', value: 'high' })
  @ModalOption({ label: 'Critical', value: 'critical' })
  priority!: string;
}
````

</Tab>
<Tab label="Grupo de Checkbox">
```typescript filename="src/components/notifications.modal.ts"
import { ModalComponent, ModalCheckboxGroup, ModalOption } from '@spraxium/components';

@ModalComponent({ id: 'notif-modal', title: 'Notification Settings' })
export class NotificationsModal {
  @ModalCheckboxGroup({
    label: 'Notification Types',
    minValues: 1,
    maxValues: 3,
  })
  @ModalOption({ label: 'Mentions', value: 'mentions' })
  @ModalOption({ label: 'Direct Messages', value: 'dms' })
  @ModalOption({ label: 'Server Events', value: 'events' })
  types!: string[];
}

````
</Tab>

## Uploads de arquivo e exibições de texto

O decorator _@ModalFileUpload_ adiciona um campo de anexo ao modal. O decorator _@ModalTextDisplay_ adiciona texto estático somente leitura que não é submetido como valor de campo. Exibições de texto são úteis para mostrar instruções, avisos ou informações de contexto dentro do próprio formulário do modal.

```typescript filename="src/components/upload-evidence.modal.ts"
import {
  ModalComponent,
  ModalTextDisplay,
  ModalFileUpload,
  ModalInput,
} from '@spraxium/components';

@ModalTextDisplay({
  content:
    'Please upload a screenshot or log file as evidence. Accepted formats: PNG, JPG, TXT.',
})
@ModalComponent({ id: 'evidence-modal', title: 'Upload Evidence' })
export class UploadEvidenceModal {
  @ModalFileUpload({
    label: 'Evidence File',
    required: true,
  })
  file!: string;

  @ModalInput({
    label: 'Additional Notes',
    style: 'paragraph',
    required: false,
    placeholder: 'Any extra context about the evidence...',
  })
  notes!: string;
}
```

```typescript filename="src/utils/modal-helpers.ts"
import { Modal } from '@spraxium/components';

export function createReasonField(required = true) {
  return Modal.field.text({
    id: 'reason',
    label: 'Reason',
    placeholder: 'Provide a reason...',
    style: 'paragraph',
    required,
    minLength: 10,
  });
}

export function createPriorityChoices() {
  return [
    Modal.choice({ label: 'Low', value: 'low' }),
    Modal.choice({ label: 'Medium', value: 'medium' }),
    Modal.choice({ label: 'High', value: 'high' }),
  ];
}
```
