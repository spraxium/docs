## @Cron e @Interval

Os dois tipos de job recorrentes cobrem a grande maioria das necessidades de agendamento. _@Cron_ executa em agendamentos baseados em calendário usando expressões cron padrão de 5 ou 6 campos. _@Interval_ repete em um offset fixo de milissegundos a partir da execução anterior. Ambos os decorators aceitam um objeto de opções para nomeação, desativação e controle de execução imediata na inicialização.

A opção _name_ é importante porque é a chave que você usa para pausar, retomar ou destruir o job em runtime através do _ScheduleService_. Se você omitir o nome, o framework gera um automaticamente a partir do nome da classe e do método, mas nomes explícitos são mais fáceis de referenciar em comandos admin e dashboards. A opção _disabled_ permite que você defina um job no código mas impeça que ele inicie, o que é útil para feature flags.

O decorator _@Cron_ também aceita uma opção _timezone_ que sobrescreve o timezone global do config de schedule. Isso é útil quando um único bot atende comunidades em fusos horários diferentes e você precisa que um resumo diário saia às 9h horário local em cada região.

<CodeTabs>
  <Tab label="Job cron" language="typescript">
```typescript filename="src/modules/tasks/services/cleanup.service.ts"
import { Injectable } from '@spraxium/common';
import { Cron, CronExpression } from '@spraxium/schedule';

@Injectable()
export class CleanupService {
  @Cron(CronExpression.EVERY_DAY_AT_MIDNIGHT, {
    name: 'cleanup-expired-warnings',
    timezone: 'UTC',
  })
  async cleanExpiredWarnings(): Promise<void> {
    // Remover avisos com mais de 30 dias
  }
}

````
  </Tab>
  <Tab label="Job de intervalo" language="typescript">
```typescript filename="src/modules/tasks/services/sync.service.ts"
import { Injectable } from '@spraxium/common';
import { Interval, IntervalExpression } from '@spraxium/schedule';

@Injectable()
export class SyncService {
  @Interval(IntervalExpression.EVERY_5_MINUTES, {
    name: 'sync-member-count', })
  async syncMemberCount(): Promise<void> {
    // Atualizar contagem de membros cacheada da API
  }
}
````

## @Timeout e @AfterOnline

Os dois tipos de job único lidam com padrões de inicialização adiada. _@Timeout_ dispara uma vez após um atraso fixo a partir do boot da aplicação. _@AfterOnline_ dispara uma vez após o client Discord emitir seu evento _ready_ mais um atraso adicional opcional. Nenhum dos jobs repete, eles executam exatamente uma vez e então o framework os remove do registry.

A diferença chave entre _@Timeout_ e _@AfterOnline_ é o ponto de partida do atraso. Um timeout começa a contar a partir do momento em que a aplicação inicia, o que significa que o client Discord pode não estar conectado ainda. Um job after-online começa a contar a partir do evento _ready_, garantindo que client, servidores e canais estão disponíveis quando o método executa. Use _@AfterOnline_ sempre que seu job precise chamar métodos da API do Discord.

Ambos os decorators aceitam as mesmas opções _name_ e _disabled_. O decorator _@Timeout_ não suporta _runOnInit_ porque é inerentemente um timer único que sempre executa uma vez. O valor de atraso é em milissegundos para ambos.

<CodeTabs>
  <Tab label="Timeout" language="typescript">
```typescript filename="src/modules/tasks/services/init.service.ts"
import { Injectable } from '@spraxium/common';
import { Timeout, TimeoutExpression } from '@spraxium/schedule';

@Injectable()
export class InitService {
  @Timeout(TimeoutExpression.AFTER_30_SECONDS, {
    name: 'deferred-cache-warmup',
  })
  async warmCache(): Promise<void> {
    // Popular caches 30 segundos após o boot
  }
}

````
  </Tab>
  <Tab label="Após online" language="typescript">
```typescript filename="src/modules/tasks/services/welcome.service.ts"
import { Injectable } from '@spraxium/common';
import { AfterOnline } from '@spraxium/schedule';

@Injectable()
export class WelcomeService {
  @AfterOnline(5000, {
    name: 'post-online-announcement', })
  async announceOnline(): Promise<void> {
    // Enviar mensagem 'bot online' 5 segundos ap\u00f3s o ready
  }
}
````

## Helpers de enum

O módulo de schedule fornece três objetos tipo-enum com constantes nomeadas para valores de temporização comuns. Usar essas constantes torna seu código mais legível do que números brutos e strings cron. Cada helper cobre os valores mais frequentes que você usaria no contexto de um bot Discord.

_CronExpression_ fornece mais de 45 presets como `EVERY_DAY_AT_MIDNIGHT`, `EVERY_HOUR`, `EVERY_MONDAY_AT_MIDNIGHT` e `EVERY_5_MINUTES`. _IntervalExpression_ fornece mais de 25 presets em milissegundos como `EVERY_MINUTE`, `EVERY_5_MINUTES` e `EVERY_HOUR`. _TimeoutExpression_ fornece mais de 20 presets como `AFTER_10_SECONDS`, `AFTER_5_MINUTES` e `AFTER_1_HOUR`.

Você sempre pode passar valores brutos ao invés de constantes de enum. Os helpers de enum são apenas re-exports de conveniência que melhoram a legibilidade e previnem erros de digitação em expressões cron.

```typescript filename="src/modules/tasks/services/example.service.ts"
import { Injectable } from '@spraxium/common';
import {
  Cron,
  CronExpression,
  Interval,
  IntervalExpression,
} from '@spraxium/schedule';

@Injectable()
export class ExampleService {
  @Cron(CronExpression.EVERY_WEEKDAY)
  async weekdayReport(): Promise<void> {}

  @Cron('0 */6 * * *')
  async customCron(): Promise<void> {}

  @Interval(IntervalExpression.EVERY_30_SECONDS)
  async frequentPoll(): Promise<void> {}
}
```
