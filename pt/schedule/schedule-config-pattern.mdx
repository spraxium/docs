## Padrão split-config

Como outros plugins do Spraxium, o módulo de schedule usa o padrão split-config onde a configuração fica em um arquivo dedicado dentro do diretório `config/`. O helper _defineSchedule_ fornece type safety completo e auto-complete para todos os campos do config. Seu `spraxium.config.ts` principal referencia o arquivo de config, mantendo o config raiz limpo e a configuração de cada plugin isolada.

Esse padrão escala bem conforme seu bot cresce. Cada plugin tem seu próprio arquivo de config com suas próprias definições de tipo, e o config raiz simplesmente importa e compõe todos. Você nunca acaba com um objeto de config monolítico que mistura configurações de prefix, schedule e HTTP em um único lugar.

O framework automaticamente descobre e carrega o config de schedule durante o bootstrap. Você não precisa chamar nenhuma função de registro manualmente. Basta exportar o config do arquivo e referenciá-lo no array de imports do config raiz.

<CodeTabs>
  <Tab label="Config de schedule" language="typescript">
```typescript filename="config/schedule.config.ts"
import { defineSchedule } from '@spraxium/schedule';

export default defineSchedule({
  timezone: 'America/Sao_Paulo',
  defaultLockTtlMs: 30_000,
  disableAll: false,
});
```
  </Tab>
  <Tab label="Config raiz" language="typescript">
```typescript filename="spraxium.config.ts"
import { defineConfig } from '@spraxium/core';
import schedule from './config/schedule.config';

export default defineConfig({
  plugins: [schedule],
});
```
  </Tab>
</CodeTabs>

## Organização do projeto

Serviços de schedule são providers injetáveis regulares que você registra em qualquer módulo. Um padrão comum é criar um módulo de tasks dedicado que agrupa todos os jobs em segundo plano, mas você também pode colocar métodos decorados com schedule em serviços específicos de domínio. A escolha depende de se seus jobs são transversais (como limpeza e sincronização) ou fortemente acoplados a uma feature específica.

Para tarefas transversais, um módulo de tasks centralizado mantém as coisas organizadas. Para tarefas específicas de feature, colocar o decorator de schedule em um método no serviço da feature evita indireção desnecessária. Ambas as abordagens são válidas e o framework as trata de forma idêntica.

## Padrões comuns

### Combinando múltiplos tipos de job em um serviço

Um único serviço pode ter métodos decorados com diferentes tipos de job. Isso é útil quando tarefas relacionadas têm requisitos de temporização diferentes mas compartilham as mesmas dependências. O scheduler trata cada método decorado como um job independente com seu próprio nome e ciclo de vida.

```typescript filename="src/modules/tasks/services/maintenance.service.ts"
import { Injectable } from "@spraxium/common";
import {
  Cron,
  CronExpression,
  Interval,
  IntervalExpression,
  AfterOnline,
} from "@spraxium/schedule";

@Injectable()
export class MaintenanceService {
  @Cron(CronExpression.EVERY_DAY_AT_3AM, { name: "daily-cleanup" })
  async dailyCleanup(): Promise<void> {
    // Arquivar logs antigos, limpar dados tempor\u00e1rios
  }

  @Interval(IntervalExpression.EVERY_10_MINUTES, { name: "health-ping" })
  async healthPing(): Promise<void> {
    // Reportar sa\u00fade do bot para servi\u00e7o de monitoramento
  }

  @AfterOnline(10_000, { name: "restore-presences" })
  async restorePresences(): Promise<void> {
    // Restaurar status do bot a partir do banco de dados ap\u00f3s reconex\u00e3o
  }
}
```

### Execução condicional de jobs com feature flags

Às vezes você quer definir um job mas só habilitá-lo baseado em variáveis de ambiente ou condições de runtime. A opção _disabled_ nas opções do decorator permite controlar isso no momento do registro. Para controle em runtime, use os métodos _ScheduleService.pause_ e _ScheduleService.resume_.

```typescript filename="src/modules/tasks/services/analytics.service.ts"
import { Injectable } from "@spraxium/common";
import { Cron, CronExpression } from "@spraxium/schedule";

@Injectable()
export class AnalyticsService {
  @Cron(CronExpression.EVERY_HOUR, {
    name: "hourly-analytics",
    disabled: process.env.ENABLE_ANALYTICS !== "true",
  })
  async collectAnalytics(): Promise<void> {
    // Coletar e agregar analytics de uso
  }
}
```

### Registro no módulo

Registre seus serviços de tarefa como providers em um módulo. O _ScheduleModule_ é global e não precisa ser importado no seu feature module. Basta adicionar seus serviços ao array de providers e o scheduler descobrirá seus métodos decorados automaticamente.

```typescript filename="src/modules/tasks/tasks.module.ts"
import { Module } from "@spraxium/common";
import { CleanupService } from "./services/cleanup.service";
import { SyncService } from "./services/sync.service";
import { MaintenanceService } from "./services/maintenance.service";

@Module({
  providers: [CleanupService, SyncService, MaintenanceService],
})
export class TasksModule {}
```
