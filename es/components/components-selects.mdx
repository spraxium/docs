## Tipos de menú de selección

Discord proporciona cinco variantes de menú de selección, y Spraxium las soporta todas a través de decorators dedicados. Los selects de string permiten definir opciones personalizadas con labels y valores. Los selects de user, role, mentionable y channel usan los selectores nativos de entidad de Discord que se rellenan automáticamente con datos del servidor. Los cinco tipos comparten el mismo patrón de handler y sistema de inyección de parámetros.

Cada tipo de select se define como una clase decorada, se vincula a un handler vía un decorator de handler específico del tipo, y se renderiza a través de _SelectService_. La diferencia principal entre los tipos es el decorator utilizado y el tipo de dato que el usuario selecciona. Los selects de string requieren definiciones de opciones explícitas, mientras que los otros cuatro tipos rellenan sus opciones desde el servidor automáticamente.

## Selects de string con opciones estáticas

Los selects de string son el tipo más flexible porque controlas las opciones directamente. Usa el decorator apilable _@SelectOption_ en la misma propiedad para definir cada opción. Cada opción tiene un _label_ (mostrado al usuario), un _value_ (enviado en la selección) y una _description_ opcional. El orden de los decorators _@SelectOption_ determina el orden de las opciones en el menú renderizado.

<CodeTabs>
<Tab label="Componente">
```typescript filename="src/components/color-picker.select.ts"
import { StringSelect, SelectOption } from '@spraxium/components';

@StringSelect({
  customId: 'color-picker',
  placeholder: 'Choose a color',
  minValues: 1,
  maxValues: 1,
})
export class ColorPickerSelect {
@SelectOption({ label: 'Red', value: 'red', description: 'A warm color' })
@SelectOption({ label: 'Blue', value: 'blue', description: 'A cool color' })
@SelectOption({ label: 'Green', value: 'green', description: 'A natural color' })
options: void;
}

````
</Tab>
<Tab label="Handler">
```typescript filename="src/components/color-picker.handler.ts"
import { StringSelectHandler, Ctx, Selected } from '@spraxium/components';
import type { StringSelectMenuInteraction } from 'discord.js';
import { ColorPickerSelect } from './color-picker.select';

@StringSelectHandler(ColorPickerSelect)
export class ColorPickerHandler {
  async handle(
    @Ctx() interaction: StringSelectMenuInteraction,
    @Selected() color: string,
  ): Promise<void> {
    await interaction.reply({
      content: `You picked ${color}.`,
      ephemeral: true,
    });
  }
}
````

</Tab>
</CodeTabs>

El objeto de configuración pasado a _@StringSelect_ controla la apariencia y el comportamiento del menú. El _customId_ y el _placeholder_ son obligatorios. Los campos _minValues_ y _maxValues_ controlan cuántas opciones el usuario puede seleccionar simultáneamente.

## Opciones dinámicas

Cuando las opciones del select dependen de datos de runtime, como registros de base de datos o configuraciones del servidor, usa la propiedad _dynamicOptions_ en el decorator _@StringSelect_. Este callback recibe un objeto de datos en tiempo de build y retorna un array de objetos de opción.

<Callout type="tip">
  Los decorators *@SelectOption* y *dynamicOptions* son mutuamente excluyentes.
  Cuando ambos están presentes en la misma clase, el callback dinámico tiene
  precedencia y las opciones estáticas se ignoran por completo.
</Callout>

```typescript filename="src/components/role-list.select.ts"
import { StringSelect } from '@spraxium/components';

@StringSelect({
  customId: 'role-list',
  placeholder: 'Pick a role',
  dynamicOptions: (roles: Array<{ id: string; name: string }>) =>
    roles.map((r) => ({
      label: r.name,
      value: r.id,
    })),
})
export class RoleListSelect {}
```

## Selects de entidad

Los selects de entidad usan los selectores nativos de Discord que muestran avatares de miembros, colores de rol e iconos de canal. No necesitan decorators _@SelectOption_ porque las opciones provienen del servidor automáticamente. Cada tipo de entidad tiene su propio par de decorators para componente y handler.

<CodeTabs>
<Tab label="Select de Usuario">

## Inyección de parámetros en handlers

Los handlers de select reciben los valores seleccionados a través de decorators de parámetro. Usa *@Selected()* para obtener el primer valor seleccionado como string, o *@SelectedValues()* para obtener todos los valores como array de strings. El decorator *@Ctx()* proporciona acceso a la interacción cruda de discord.js para escenarios que necesitan más control.

```typescript filename="src/components/multi-role.handler.ts"
import { UserSelectHandler, Ctx, SelectedValues } from '@spraxium/components';
import type { UserSelectMenuInteraction } from 'discord.js';
import { AssignUserSelect } from './assign-user.select';

@UserSelectHandler(AssignUserSelect)
export class AssignUserHandler {
  async handle(
    @Ctx() interaction: UserSelectMenuInteraction,
    @SelectedValues() userIds: string[],
  ): Promise<void> {
    await interaction.reply({
      content: `Selected ${userIds.length} user(s).`,
      ephemeral: true,
    });
  }
}
````

## Renderizado con SelectService

El _SelectService_ transforma metadatos de decorator en action rows de discord.js. El método _build_ acepta la clase del select y datos opcionales (para _dynamicOptions_). El método _buildWithContext_ adjunta un ID de flow context al _customId_ del select.

```typescript filename="src/commands/pick-color.command-handler.ts"
import { Ctx, SlashCommandHandler } from '@spraxium/common';
import { SelectService } from '@spraxium/components';
import type { ChatInputCommandInteraction } from 'discord.js';
import { ColorPickerSelect } from '../components/color-picker.select';
import { PickColorCommand } from './pick-color.command';

@SlashCommandHandler(PickColorCommand)
export class PickColorCommandHandler {
  constructor(private readonly selects: SelectService) {}

  async handle(@Ctx() interaction: ChatInputCommandInteraction): Promise<void> {
    const row = await this.selects.build(ColorPickerSelect);

    await interaction.reply({
      content: 'Pick your favorite color:',
      components: [row],
    });
  }
}
```

## Constructor fluente alternativo

Para escenarios donde necesitas construir definiciones de select programáticamente, usa el constructor fluente _Selects.string()_. Este constructor produce la misma salida que los decorators, pero a través de una API de encadenamiento de métodos. Es útil en funciones utilitarias o cuando la configuración del select depende enteramente de datos de runtime.

```typescript filename="src/utils/select-factory.ts"
import { Selects } from '@spraxium/components';

export function buildCategorySelect(categories: string[]) {
  return Selects.string('category-select')
    .placeholder('Pick a category')
    .options(categories.map((c) => ({ label: c, value: c.toLowerCase() })))
    .build();
}
```

## Registro en módulo

Solo los handlers necesitan registrarse, en el array _handlers_ de un módulo de feature. Las clases de componente de select decoradas con _@StringSelect_, _@UserSelect_ o similares son solo metadatos y nunca se agregan a ningún módulo. El escáner de instancias identifica selects por el metadato del decorator en el handler y los registra en el _SelectDispatcher_ durante la fase de arranque.

```typescript filename="src/components/components.module.ts"
import { Module } from '@spraxium/core';
import { ColorPickerHandler } from './color-picker.handler';
import { AssignUserHandler } from './assign-user.handler';

@Module({
  handlers: [ColorPickerHandler, AssignUserHandler],
})
export class SelectsModule {}
```
