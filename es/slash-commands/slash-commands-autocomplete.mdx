## Qué es autocomplete

Autocomplete es un tipo de interacción de Discord que se dispara cada vez que un usuario escribe en una opción de comando marcada con `autocomplete: true`. En vez de esperar a que el usuario envíe el comando completo, Discord envía una *AutocompleteInteraction* que contiene el texto parcial que el usuario ha escrito hasta el momento. El bot responde con hasta 25 sugerencias que Discord muestra como dropdown debajo del campo de entrada. El usuario puede seleccionar una sugerencia o continuar escribiendo, y Discord envía nuevas interacciones de autocomplete conforme la entrada cambia.

En Spraxium, el autocomplete se maneja con una clase handler dedicada decorada con *@SlashAutocompleteHandler*. Este handler es separado del handler de comando porque las interacciones de autocomplete llegan antes de que el comando se envíe y tienen un tipo de interacción diferente. El handler de autocomplete recibe la entrada parcial a través del decorador de parámetro *@SlashFocused* y responde con sugerencias filtradas a través del objeto *AutocompleteInteraction* inyectado por *@Ctx*.

El sistema de autocomplete se integra con el mismo contenedor DI, registro de módulos y manejo de excepciones que los handlers de comando. Las clases handler de autocomplete pueden inyectar servicios para obtener sugerencias de bases de datos, APIs o fuentes de datos en caché. Los errores durante la evaluación de autocomplete son capturados por el exception handler y no hacen caer el proceso del bot.

## Habilitando autocomplete en una opción

El autocomplete se habilita del lado de la definición del comando configurando `autocomplete: true` en los parámetros de la opción. Este flag le dice a Discord que envíe interacciones de autocomplete cuando el usuario enfoque el campo de la opción. Solo las opciones *String*, *Integer* y *Number* soportan autocomplete. Cuando *autocomplete* es `true`, la propiedad *choices* se ignora porque Discord no permite choices estáticos y autocomplete dinámico en la misma opción.

```typescript filename="commands/search.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';

@SlashCommand({ name: 'search', description: 'Search for an item in the database.' })
export class SearchCommand {
  @SlashOption.String('query', {
    description: 'Search query', required: true, autocomplete: true, })
  build() {}
}
```

## La clase handler de autocomplete

El decorador *@SlashAutocompleteHandler* marca una clase como proveedor de autocomplete para una opción específica en un comando específico. Acepta dos argumentos: la clase *@SlashCommand* dueña de la opción, y el nombre de la opción como string. La clase debe exponer un método *handle* que recibe el valor enfocado y la interacción, procesa la entrada y responde con sugerencias.

```typescript filename="handlers/search.autocomplete.ts"
import { Ctx, SlashAutocompleteHandler, SlashFocused } from '@spraxium/common';
import type { AutocompleteInteraction } from 'discord.js';
import { SearchCommand } from '../commands/search.command';

@SlashAutocompleteHandler(SearchCommand, 'query')
export class SearchAutocomplete {
  async handle(
    @SlashFocused() query: string, @Ctx() interaction: AutocompleteInteraction, ): Promise<void> {
    const results = await this.fetchSuggestions(query);
    await interaction.respond(
      results.map((r) => ({ name: r.label, value: r.id })), );
  }

  private async fetchSuggestions(query: string) {
    return [
      { label: 'Item One', id: 'item-1' }, { label: 'Item Two', id: 'item-2' }, ].filter((item) => item.label.toLowerCase().includes(query.toLowerCase()));
  }
}
```

El decorador *@SlashAutocompleteHandler* recibe la referencia de la clase de comando como primer argumento y el nombre de la opción como segundo. Durante la carga de módulos, el framework lee estos metadatos y empareja el handler con la opción exacta a la que debe responder. Cada handler está vinculado a un comando y a una opción, por lo que debes crear una clase separada para cada opción que necesite autocomplete.

## Inyección de parámetros en handlers de autocomplete

Los handlers de autocomplete usan dos decoradores de parámetro: *@SlashFocused()* y *@Ctx()*. El decorador *@SlashFocused* inyecta el valor parcial que el usuario ha escrito hasta el momento. Para opciones *String* esto es un string, para opciones *Integer* y *Number* puede ser un string o número dependiendo del estado actual de la entrada. El decorador *@Ctx* inyecta el objeto *AutocompleteInteraction*, que proporciona el método *respond* para enviar sugerencias de vuelta a Discord.

El orden de los parámetros no importa. El *SlashBinder* lee los índices de metadatos de ambos decoradores y construye el array de parámetros posicionalmente. Si colocas *@SlashFocused* en el índice 0 y *@Ctx* en el índice 1, o los inviertes, el framework inyecta los valores correctos en las posiciones correctas.

<Callout tone="warning" title="Responde en menos de tres segundos">
  Discord exige que las respuestas de autocomplete lleguen dentro de los tres segundos posteriores al envío de la interacción.
  Si tu handler supera el límite de tiempo, el cliente Discord muestra una lista vacía o con error y el usuario no puede seleccionar ninguna opción.
</Callout>

## Patrones de filtrado y respuesta

Discord impone un máximo de 25 sugerencias por respuesta de autocomplete. Tu handler debe filtrar y recortar los resultados antes de llamar a *interaction.respond()*. Cada sugerencia es un objeto con *name* (mostrado al usuario, máximo 100 caracteres) y *value* (enviado al handler de comando cuando se selecciona).

<CodeTabs defaultIndex={0}>
  <Tab label="Filtro de lista estática" language="typescript">
```typescript filename="handlers/ping-service.autocomplete.ts"
import { Ctx, SlashAutocompleteHandler, SlashFocused } from '@spraxium/common';
import type { AutocompleteInteraction } from 'discord.js';
import { PingCommand } from '../commands/ping.command';

const SERVICES = [
  { name: 'Gateway', value: 'gateway' }, { name: 'REST API', value: 'rest-api' }, { name: 'Voice', value: 'voice' }, { name: 'CDN', value: 'cdn' }, { name: 'Database', value: 'database' }, ];

@SlashAutocompleteHandler(PingCommand, 'service')
export class PingServiceAutocomplete {
  async handle(
    @SlashFocused() focused: string, @Ctx() interaction: AutocompleteInteraction, ): Promise<void> {
    const filtered = SERVICES.filter((s) =>
      s.name.toLowerCase().includes(String(focused).toLowerCase()), );
    await interaction.respond(filtered.slice(0, 25));
  }
}
```
  </Tab>
  <Tab label="Filtro con base de datos" language="typescript">
```typescript filename="handlers/tag-search.autocomplete.ts"
import { Ctx, SlashAutocompleteHandler, SlashFocused } from '@spraxium/common';
import type { AutocompleteInteraction } from 'discord.js';
import { TagCommand } from '../commands/tag.command';
import { TagService } from '../../services/tag.service';

@SlashAutocompleteHandler(TagCommand, 'name')
export class TagSearchAutocomplete {
  constructor(private readonly tags: TagService) {}

  async handle(
    @SlashFocused() focused: string, @Ctx() interaction: AutocompleteInteraction, ): Promise<void> {
    const guildId = interaction.guildId ?? '';
    const matches = await this.tags.search(guildId, String(focused), 25);
    await interaction.respond(
      matches.map((tag) => ({ name: tag.name, value: tag.id })), );
  }
}
```

## Autocomplete con inyección de dependencias

Los handlers de autocomplete se instancian a través del contenedor DI, así que pueden inyectar servicios exactamente como los handlers de comando. Esto habilita patrones donde las sugerencias de autocomplete vienen de una base de datos, caché o API externa. La clase handler declara parámetros de constructor, y el contenedor DI los resuelve de los providers del módulo durante la carga.

```typescript filename="handlers/song-search.autocomplete.ts"
import { Ctx, SlashAutocompleteHandler, SlashFocused } from '@spraxium/common';
import type { AutocompleteInteraction } from 'discord.js';
import { PlayCommand } from '../commands/play.command';
import { MusicService } from '../../services/music.service';

@SlashAutocompleteHandler(PlayCommand, 'song')
export class SongSearchAutocomplete {
  constructor(private readonly music: MusicService) {}

  async handle(
    @SlashFocused() focused: string, @Ctx() interaction: AutocompleteInteraction, ): Promise<void> {
    const results = await this.music.searchTracks(String(focused));
    await interaction.respond(
      results.slice(0, 25).map((track) => ({
        name: `${track.artist} - ${track.title}`.slice(0, 100), value: track.uri, })), );
  }
}
```

## Registro en módulo para handlers de autocomplete

Los handlers de autocomplete se registran en el array *handlers* del módulo junto con los handlers de comando. El framework detecta los metadatos de *@SlashAutocompleteHandler* durante la carga del módulo y conecta el handler a su comando y opción de forma automática. No necesitas arrays separados o configuración especial para handlers de autocomplete.

```typescript filename="modules/slash/slash.module.ts"
import { Module } from '@spraxium/common';
import { PingCommand } from './commands/ping.command';
import { PingHandler } from './handlers/ping.handler';
import { PingServiceAutocomplete } from './handlers/ping-service.autocomplete';

@Module({
  commands: [PingCommand], handlers: [PingHandler, PingServiceAutocomplete], })
export class SlashModule {}
```

## Checklist de diseño para autocomplete

<Steps>
  <Step title="Siempre recorta a 25 resultados">
    Discord rechaza respuestas de autocomplete con más de 25 opciones. Aplica `.slice(0, 25)` a todo array de respuesta, incluso si esperas menos resultados. Esto previene fallos inesperados cuando los datos crecen más allá de tus expectativas iniciales.
  </Step>
  <Step title="Trunca nombres de sugerencia a 100 caracteres">
    Discord limita el campo *name* a 100 caracteres. Cuando las sugerencias vienen de contenido generado por usuarios o fuentes externas, aplica `.slice(0, 100)` al nombre para prevenir rechazo de la API.
  </Step>
  <Step title="Convierte el valor enfocado a string antes de filtrar">
    Para opciones *Integer* y *Number*, el valor enfocado puede llegar como número o string parcial que aún no es un número válido. Usa `String(focused)` antes de operaciones con string como *toLowerCase()* o *includes()* para evitar errores de tipo.
  </Step>
  <Step title="Responde rápido con datos en caché o pre-computados">
    Discord espera respuestas de autocomplete dentro de tres segundos. Para sugerencias respaldadas por base de datos, considera cachear consultas frecuentes o usar índices de búsqueda optimizados. Para listas estáticas, define los datos como constante a nivel de módulo para evitar asignaciones repetidas.
  </Step>
  <Step title="Retorna array vacío cuando no exista handler">
    Si una opción de comando tiene `autocomplete: true` pero no hay handler registrado, el binder envía un array vacío automáticamente. Sin embargo, si creas un handler que falla al responder, el usuario ve un estado de carga perpetuo. Siempre llama a `interaction.respond()` en cada camino de código, incluso con un array vacío.
  </Step>
</Steps>
