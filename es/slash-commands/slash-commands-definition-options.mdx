## La clase de definición de comando

Una definición de slash command es una clase decorada con *@SlashCommand* que declara lo que Discord debe mostrar en el selector de comandos. La clase en sí no contiene lógica de ejecución. Su propósito es almacenar metadatos que el *SlashRegistrar* lee al inicio para construir el payload JSON enviado a la API REST de Discord. Las opciones se declaran como decoradores de método en un método *build()*, y los subcomandos se declaran en métodos nombrados. La clase puede ser importada por handlers, handlers de autocomplete y fixtures de test sin arrastrar dependencias de servicios.

El decorador *@SlashCommand* acepta un objeto *SlashCommandConfig* con las siguientes propiedades. Cada comando debe proporcionar *name* y *description* como mínimo. Las propiedades restantes controlan permisos por defecto, disponibilidad en DMs, marcado NSFW, registro específico de guild y localización.

<DropdownGroup>
  <Dropdown title="name, string (obligatorio)">
    El nombre del comando mostrado en el selector de Discord. Debe ser minúscula, entre 1 y 32 caracteres, sin espacios. Es el identificador que Discord usa para enrutar la interacción a tu bot.
    Cambiar el nombre de un comando después del registro crea una nueva entrada en el registry de Discord; el nombre anterior no se elimina automáticamente hasta que fuerces un nuevo registro.
  </Dropdown>
  <Dropdown title="description, string (obligatorio)">
    La descripción mostrada debajo del nombre del comando en el selector de Discord. Acepta de 1 a 100 caracteres. Escribe una descripción concisa y orientada a la acción para que los usuarios entiendan el propósito del comando antes de invocarlo.
    Discord indexa las descripciones para búsqueda, por lo que textos claros mejoran la visibilidad del comando.
  </Dropdown>
  <Dropdown title="defaultMemberPermissions, bigint | number | null">
    Bitfield de permisos requeridos para ver y usar el comando. Usa `PermissionFlagsBits` de discord.js para componer el valor. Cuando se configura, Discord oculta el comando a los miembros que no tienen los permisos especificados.
    Configura `null` para hacer el comando visible a todos por defecto. Los administradores del servidor pueden sobrescribir esto por guild en la configuración del servidor.
  </Dropdown>
  <Dropdown title="dmPermission, boolean">
    Controla si el comando está disponible en mensajes directos. Por defecto es `true` cuando se omite. Configura `false` para comandos que requieren contexto de guild, como los que leen miembros o canales.
    Esta flag no tiene efecto en comandos específicos de guild creados mediante la propiedad *guild*.
  </Dropdown>
  <Dropdown title="nsfw, boolean">
    Marca el comando como restringido por edad. Cuando es `true`, el comando solo aparece en canales NSFW y en DMs para usuarios que han reconocido las restricciones de edad.
    Usa esta flag para comandos que presentan contenido para adultos, en cumplimiento con las políticas de Discord.
  </Dropdown>
  <Dropdown title="guild, string">
    ID de guild para registro específico de guild. Cuando se proporciona, el comando solo se registra en esa guild y omite el registro global. Los comandos de guild se propagan en segundos en lugar del retraso de hasta una hora de los comandos globales.
    Elimina la propiedad *guild* antes de desplegar en producción para registrar el comando globalmente.
  </Dropdown>
  <Dropdown title="i18n, SlashI18nKeys">
    Claves de localización para el nombre y descripción del comando, resueltas a través del módulo i18n. Proporciona un objeto *SlashI18nKeys* con las claves *name* y *description* que mapean a entradas en tus archivos de locale.
    Los campos base *name* y *description* sirven como fallback cuando no se encuentra un locale correspondiente para el idioma del cliente Discord del usuario.
  </Dropdown>
</DropdownGroup>

```typescript filename="commands/ban.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';
import { PermissionFlagsBits } from 'discord.js';

@SlashCommand({
  name: 'ban', description: 'Ban a member from the server.', defaultMemberPermissions: PermissionFlagsBits.BanMembers, })
export class BanCommand {
  @SlashOption.User('target', { description: 'The member to ban', required: true })
  @SlashOption.String('reason', { description: 'Reason for the ban', required: false, maxLength: 512 })
  @SlashOption.Integer('days', {
    description: 'Number of days of messages to delete', required: false, min: 0, max: 7, })
  build() {}
}
```

## Tipos de opciones

Spraxium proporciona nueve decoradores de tipo de opción a través del namespace *SlashOption*, uno para cada tipo de opción de Application Command de Discord. Cada decorador es un decorador de método aplicado al método *build()* de la clase de comando (o a un método *@SlashSubcommand* cuando la opción pertenece a un subcomando). Las opciones se apilan en el mismo método, y el orden de los decoradores determina el orden de visualización en el cliente Discord (el decorador superior aparece último en el selector).

<DropdownGroup>
  <Dropdown title="@SlashOption.String, string | null">
    Mapea al tipo STRING de Discord. El handler recibe `string` cuando el usuario proporciona un valor, o `null` cuando la opción no es obligatoria y fue omitida.
    Soporta *minLength*, *maxLength*, *choices* (lista estática) y *autocomplete* (sugerencias dinámicas). Ten en cuenta que *choices* y *autocomplete* no pueden combinarse en la misma opción.
  </Dropdown>
  <Dropdown title="@SlashOption.Integer, number | null">
    Mapea al tipo INTEGER de Discord. Acepta solo números enteros. El handler recibe `number` o `null`.
    Soporta *min* y *max* para validación de rango, *choices* para un conjunto fijo de enteros y *autocomplete* para sugerencias dinámicas.
  </Dropdown>
  <Dropdown title="@SlashOption.Number, number | null">
    Mapea al tipo NUMBER de Discord. Acepta números de punto flotante. El handler recibe `number` o `null`.
    Soporta *min* y *max* para validación de rango y *autocomplete* para sugerencias dinámicas. Los *choices* estáticos no están disponibles para el tipo Number.
  </Dropdown>
  <Dropdown title="@SlashOption.Boolean, boolean | null">
    Mapea al tipo BOOLEAN de Discord. Discord lo renderiza como un toggle verdadero/falso en el selector de comandos. El handler recibe `true`, `false` o `null` cuando la opción es opcional y no fue completada.
    No hay restricciones adicionales disponibles para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.User, User | null">
    Mapea al tipo USER de Discord. Discord renderiza un selector de usuario. El handler recibe un objeto *User* completamente resuelto de discord.js, o `null` si la opción es opcional y no fue completada.
    No hay restricciones adicionales disponibles para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.Channel, Channel | null">
    Mapea al tipo CHANNEL de Discord. Discord renderiza un selector de canal. El handler recibe un objeto *Channel* o `null`.
    Soporta el array *channelTypes* para restringir qué tipos de canal puede seleccionar el usuario. Consulta la sección de filtrado de tipos de canal más abajo para la lista completa de valores aceptados.
  </Dropdown>
  <Dropdown title="@SlashOption.Role, Role | null">
    Mapea al tipo ROLE de Discord. Discord renderiza un selector de rol. El handler recibe un objeto *Role* completamente resuelto de discord.js, o `null` si la opción es opcional y no fue completada.
    No hay restricciones adicionales disponibles para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.Mentionable, User | Role | null">
    Mapea al tipo MENTIONABLE de Discord. Acepta tanto usuarios como roles. El handler recibe un objeto *User* o *Role* según lo seleccionado, o `null` si es opcional y fue omitido.
    No hay restricciones adicionales disponibles para este tipo.
  </Dropdown>
  <Dropdown title="@SlashOption.Attachment, Attachment | null">
    Mapea al tipo ATTACHMENT de Discord. Discord renderiza un campo de carga de archivos. El handler recibe un objeto *Attachment* de discord.js con metadatos y URL de CDN, o `null` si la opción es opcional y no fue completada.
    No hay restricciones adicionales disponibles para este tipo.
  </Dropdown>
</DropdownGroup>

Cada decorador de opción acepta dos argumentos posicionales: el *name* de la opción (string, usado para el registro en Discord y la correspondencia con *@SlashOpt*) y un objeto *config*. El objeto config extiende *SlashBaseOptionConfig* con campos específicos del tipo.

## Restricciones de opciones de texto

El decorador *@SlashOption.String* soporta cuatro propiedades de configuración adicionales más allá de la config base. Estas propiedades mapean directamente a las restricciones de la API de Discord para opciones de texto y te permiten controlar la longitud de entrada, proporcionar choices fijos o habilitar sugerencias de autocomplete.

```typescript filename="commands/echo.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';

@SlashCommand({ name: 'echo', description: 'Repeat a message back to the channel.' })
export class EchoCommand {
  @SlashOption.String('message', {
    description: 'Text to echo', required: true, minLength: 1, maxLength: 2000, })
  build() {}
}
```

Cuando se configura *minLength* o *maxLength*, Discord valida la longitud de entrada del lado del cliente antes de enviar la interacción. Cuando se proporciona *choices*, el usuario selecciona de un dropdown en vez de escribir libremente. Cuando *autocomplete* se configura a `true`, Discord llama a tu handler de autocomplete mientras el usuario escribe, proporcionando sugerencias dinámicas en tiempo real.

<Callout tone="warning" title="choices y autocomplete son mutuamente exclusivos">
  Configurar *autocomplete* a true y proporcionar un array *choices* en la misma opción no está permitido por la API de Discord. Cuando *autocomplete* está activo, los choices estáticos son ignorados y solo se consulta tu handler de autocomplete.
  Usa *choices* estáticos para listas pequeñas y fijas, y *autocomplete* para sugerencias dinámicas o provenientes de base de datos que superen las 25 entradas.
</Callout>

## Restricciones de opciones numéricas

Tanto *@SlashOption.Integer* como *@SlashOption.Number* soportan las propiedades *min* y *max* que establecen el rango de valores permitidos. Discord impone estas restricciones del lado del cliente, así que el handler tiene garantizado recibir un valor dentro del rango especificado (o `null` si la opción no es obligatoria y el usuario la omite). La variante *Integer* también soporta *choices* y *autocomplete*, mientras que la variante *Number* soporta solo *autocomplete*.

<CodeTabs defaultIndex={0}>
  <Tab label="Integer con choices" language="typescript">
```typescript filename="commands/roll.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';

@SlashCommand({ name: 'roll', description: 'Roll a dice.' })
export class RollCommand {
  @SlashOption.Integer('sides', {
    description: 'Number of sides on the dice', required: true, choices: [
      { name: '6-sided', value: 6 }, { name: '12-sided', value: 12 }, { name: '20-sided', value: 20 }, ], })
  build() {}
}
```
  </Tab>

## Filtrado de tipos de canal

El decorador *@SlashOption.Channel* acepta un array *channelTypes* que restringe qué tipos de canal puede seleccionar el usuario. Los valores corresponden a miembros del enum *ChannelType* de Discord. Cuando se proporciona el array, el cliente Discord solo muestra canales que coincidan con los tipos especificados en el selector de opciones.

```typescript filename="commands/setup.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';
import { ChannelType } from 'discord.js';

@SlashCommand({ name: 'setup', description: 'Configure the bot for this server.' })
export class SetupCommand {
  @SlashOption.Channel('log-channel', {
    description: 'Channel for bot logs', required: true, channelTypes: [ChannelType.GuildText, ChannelType.GuildAnnouncement], })
  @SlashOption.Channel('voice-channel', {
    description: 'Default voice channel', required: false, channelTypes: [ChannelType.GuildVoice, ChannelType.GuildStageVoice], })
  build() {}
}
```

## Localización con claves i18n

Tanto el decorador *@SlashCommand* como todos los decoradores *@SlashOption* aceptan una propiedad *i18n* opcional de tipo *SlashI18nKeys*. Este objeto contiene dos claves string, *name* y *description*, que el módulo i18n resuelve a cadenas específicas de locale durante el registro del comando. Las localizaciones resueltas se envían a Discord como parte del payload del comando, así que los usuarios ven nombres de comando y opciones en el idioma de su cliente Discord.

```typescript filename="commands/locale.command.ts"
import { SlashCommand, SlashOption } from '@spraxium/common';

@SlashCommand({
  name: 'locale', description: 'Change your preferred language.', i18n: {
    name: 'commands.locale.name', description: 'commands.locale.description', }, })
export class LocaleCommand {
  @SlashOption.String('language', {
    description: 'The language to use.', required: true, i18n: {
      name: 'commands.locale.options.language.name', description: 'commands.locale.options.language.description', }, choices: [
      { name: 'English (US)', value: 'en-US' }, { name: 'Português (Brasil)', value: 'pt-BR' }, ], })
  build() {}
}
```

Las claves *i18n* siguen el mismo patrón de resolución que el resto del sistema i18n del framework. Mapean a claves en tus archivos JSON de locale y soportan todos los locales soportados por Discord. Cuando un usuario con un cliente Discord en portugués brasileño abre el selector de comandos, ve los nombres de comando y opción traducidos automáticamente.

## Checklist de diseño para definiciones de comandos

<Steps>
  <Step title="Mantén las definiciones libres de lógica de negocio">
    La clase de comando debe contener solo decoradores de metadatos y un método *build()* vacío (o métodos de subcomando nombrados). Nunca importes servicios, repositorios o dependencias de runtime en un archivo de definición de comando. Esto asegura que la clase pueda ser importada por handlers, handlers de autocomplete y tests sin crear cadenas de dependencia.
  </Step>
  <Step title="Usa el tipo de opción más específico disponible">
    Prefiere *@SlashOption.User* sobre *@SlashOption.String* para IDs de usuario, *@SlashOption.Channel* sobre *@SlashOption.String* para referencias de canal, y *@SlashOption.Integer* sobre *@SlashOption.Number* para números enteros. Los tipos específicos le dan a Discord mejores affordances de UI como selectores de usuario y dropdowns de canal, y el handler recibe objetos discord.js completamente resueltos en vez de strings crudos.
  </Step>
  <Step title="Restringe canales con channelTypes">
    Cuando un comando espera un tipo específico de canal, siempre proporciona el array *channelTypes*. Esto previene que usuarios seleccionen canales de voz donde esperas canales de texto, y el cliente Discord impone la restricción antes de que la interacción llegue a tu bot.
  </Step>
  <Step title="Prefiere choices sobre autocomplete para conjuntos pequeños">
    Los arrays de *choices* estáticos se muestran como dropdown y no requieren código de handler. Úsalos cuando el conjunto de valores válidos es pequeño y fijo (menos de 25 entradas). Usa *autocomplete* cuando los valores son dinámicos, vienen de una base de datos o exceden el límite de 25 choices.
  </Step>
  <Step title="Configura permisos a nivel de definición">
    Usa *defaultMemberPermissions* para establecer los permisos mínimos requeridos para ver el comando en el selector. Esta es una pista de UI que Discord impone del lado del cliente. Combínala con *@UseGuards* en el handler para imposición del lado del servidor que no puede ser evadida.
  </Step>
</Steps>
