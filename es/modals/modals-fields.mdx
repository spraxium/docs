## Vista general de tipos de campo

Los modales de Spraxium soportan 11 tipos de campo, cada uno declarado a través de un decorator a nivel de propiedad en tu clase _@ModalComponent_. Los inputs de texto y selects son los más comunes, pero el sistema también incluye grupos de radio, checkboxes, subida de archivos y textos estáticos de solo lectura. Cada decorator de campo acepta un objeto de configuración con propiedades específicas del tipo de campo, además de propiedades compartidas como _label_ y _required_.

El orden de los campos en el modal renderizado coincide con el orden de declaración de las propiedades decoradas en la clase. Los modales de Discord tienen un máximo de 5 action rows, y cada campo ocupa una row. Este límite significa que un solo modal puede contener como máximo 5 campos, aunque los campos dinámicos y condicionales pueden alterar lo que aparece para cada usuario.

## Inputs de texto

El decorator _@ModalInput_ crea un campo de input de texto. La propiedad _style_ controla si el campo se renderiza como un input de línea única (_short_) o multilínea (_paragraph_). Los campos cortos son ideales para nombres, asuntos y valores breves. Los campos de párrafo se adaptan a descripciones, mensajes y contenido más extenso.

```typescript filename="src/components/report.modal.ts"
import { ModalComponent, ModalInput } from '@spraxium/components';

@ModalComponent({ id: 'report-modal', title: 'Submit Report' })
export class ReportModal {
  @ModalInput({
    label: 'Title',
    placeholder: 'A short title for the report',
    required: true,
    minLength: 5,
    maxLength: 100,
    style: 'short',
  })
  title!: string;

  @ModalInput({
    label: 'Description',
    placeholder: 'Describe the issue in detail...',
    required: true,
    minLength: 20,
    maxLength: 4000,
    style: 'paragraph',
  })
  description!: string;

  @ModalInput({
    label: 'Reproduction Steps',
    placeholder: 'Optional steps to reproduce',
    required: false,
    style: 'paragraph',
    value: "1. \n2. \n3. ",
  })
  steps!: string;
}
```

## Selects de string con opciones

El decorator _@ModalSelect_ crea un menú desplegable de selección dentro del modal. Define las opciones disponibles usando el decorator apilable _@ModalChoice_ en la misma propiedad. Cada opción tiene un _label_ (mostrado al usuario), _value_ (enviado al enviar), y un _description_ opcional.

```typescript filename="src/components/category-select.modal.ts"
import { ModalComponent, ModalSelect, ModalChoice } from '@spraxium/components';

@ModalComponent({ id: 'ticket-modal', title: 'Create Ticket' })
export class TicketModal {
  @ModalSelect({
    label: 'Category',
    placeholder: 'Select a ticket category',
    required: true,
  })
  @ModalChoice({
    label: 'Bug Report',
    value: 'bug',
    description: 'Something is broken',
  })
  @ModalChoice({
    label: 'Feature Request',
    value: 'feature',
    description: 'Suggest an improvement',
  })
  @ModalChoice({
    label: 'General Question',
    value: 'question',
    description: 'Ask a question',
  })
  category!: string;

  @ModalInput({
    label: 'Details',
    style: 'paragraph',
    required: true,
  })
  details!: string;
}
```

## Selects de entidad

Los decorators de select de entidad permiten a los usuarios elegir miembros del servidor, roles, canales o entidades mencionables directamente desde el modal. Estos campos usan los selectores nativos de entidad de Discord, por lo que muestran avatares, colores de rol e iconos de canal. No se necesitan decorators _@ModalChoice_ porque las opciones se llenan desde el servidor.

<CodeTabs>
<Tab label="Select de usuario">
```typescript filename="src/components/assign.modal.ts"
import { ModalComponent, ModalUserSelect } from '@spraxium/components';

@ModalComponent({ id: 'assign-modal', title: 'Assign User' })
export class AssignModal {
  @ModalUserSelect({
    label: 'Assignee',
    placeholder: 'Pick a team member',
    required: true,
  })
  assignee!: string;
}

````
</Tab>
<Tab label="Select de canal">
```typescript filename="src/components/move-channel.modal.ts"
import { ModalComponent, ModalChannelSelect } from '@spraxium/components';
import { ChannelType } from 'discord.js';

@ModalComponent({ id: 'move-modal', title: 'Move to Channel' })
export class MoveChannelModal {
  @ModalChannelSelect({
    label: 'Target Channel',
    placeholder: 'Select a text channel',
    required: true,
    channelTypes: [ChannelType.GuildText],
  })
  channel!: string;
}
````

</Tab>
<Tab label="Select de rol">
```typescript filename="src/components/role-assign.modal.ts"
import { ModalComponent, ModalRoleSelect } from '@spraxium/components';

export class RoleAssignModal {

    required: true,

````

## Grupos de radio y checkboxes

Los grupos de radio presentan opciones mutuamente excluyentes donde el usuario debe elegir exactamente una. Los grupos de checkbox permiten selecciones múltiples. Un *@ModalCheckbox* individual se renderiza como un toggle booleano. Todos los campos basados en opciones usan *@ModalOption* (un alias de *@ModalChoice*) para definir sus opciones.

<CodeTabs>
<Tab label="Grupo de radio">
```typescript filename="src/components/priority.modal.ts"
import { ModalComponent, ModalRadioGroup, ModalOption } from '@spraxium/components';

@ModalComponent({ id: 'priority-modal', title: 'Set Priority' })
export class PriorityModal {
  @ModalRadioGroup({ label: 'Priority Level' })
  @ModalOption({ label: 'Low', value: 'low' })
  @ModalOption({ label: 'Medium', value: 'medium' })
  @ModalOption({ label: 'High', value: 'high' })
  @ModalOption({ label: 'Critical', value: 'critical' })
  priority!: string;
}
````

</Tab>
<Tab label="Grupo de checkbox">
```typescript filename="src/components/notifications.modal.ts"
import { ModalComponent, ModalCheckboxGroup, ModalOption } from '@spraxium/components';

@ModalComponent({ id: 'notif-modal', title: 'Notification Settings' })
export class NotificationsModal {
  @ModalCheckboxGroup({
    label: 'Notification Types',
    minValues: 1,
    maxValues: 3,
  })
  @ModalOption({ label: 'Mentions', value: 'mentions' })
  @ModalOption({ label: 'Direct Messages', value: 'dms' })
  @ModalOption({ label: 'Server Events', value: 'events' })
  types!: string[];
}

````
</Tab>
<Tab label="Checkbox individual">
```typescript filename="src/components/consent.modal.ts"

  agreed!: string;

  @ModalInput({ label: 'Signature', style: 'short', required: true })

}
````

## Subida de archivos y textos de visualización

El decorator _@ModalFileUpload_ agrega un campo de adjunto al modal. El decorator _@ModalTextDisplay_ agrega texto estático de solo lectura que no se envía como valor de campo. Los textos de visualización son útiles para mostrar instrucciones, avisos o información de contexto dentro del propio formulario del modal.

```typescript filename="src/components/upload-evidence.modal.ts"
import {
  ModalComponent,
  ModalTextDisplay,
  ModalFileUpload,
  ModalInput,
} from '@spraxium/components';

@ModalTextDisplay({
  content:
    'Please upload a screenshot or log file as evidence. Accepted formats: PNG, JPG, TXT.',
})
@ModalComponent({ id: 'evidence-modal', title: 'Upload Evidence' })
export class UploadEvidenceModal {
  @ModalFileUpload({
    label: 'Evidence File',
    required: true,
  })
  file!: string;

  @ModalInput({
    label: 'Additional Notes',
    style: 'paragraph',
    required: false,
    placeholder: 'Any extra context about the evidence...',
  })
  notes!: string;
}
```

```typescript filename="src/utils/modal-helpers.ts"
import { Modal } from '@spraxium/components';

export function createReasonField(required = true) {
  return Modal.field.text({
    id: 'reason',
    label: 'Reason',
    placeholder: 'Provide a reason...',
    style: 'paragraph',
    required,
    minLength: 10,
  });
}

export function createPriorityChoices() {
  return [
    Modal.choice({ label: 'Low', value: 'low' }),
    Modal.choice({ label: 'Medium', value: 'medium' }),
    Modal.choice({ label: 'High', value: 'high' }),
  ];
}
```
