## Vision general

El repositorio de Spraxium incluye varias apps en `apps/`. Cada una muestra un conjunto concreto de recursos en un proyecto pequeno, legible y facil de ejecutar localmente.

La documentacion explica la teoria. Los ejemplos muestran la practica.

Repositorio: [github.com/spraxium/spraxium/tree/main/apps](https://github.com/spraxium/spraxium/tree/main/apps)

## Mapa de ejemplos

### slash-bot

**Para que sirve:** slash commands, subcommands, guards y autocomplete.

**Como usarlo:** empieza por este si tu bot sera command-driven. Revisa `simple`, luego `subcommands`, y despues `protection`.

### modal-bot

**Para que sirve:** formularios con modal y procesamiento de submit.

**Como usarlo:** revisa `profile`, `suggestion` y `ticket` para entender apertura y manejo de modales.

### components-bot

**Para que sirve:** botones, selects, modales, flow context y layouts V2.

**Como usarlo:** usalo como referencia general de `@spraxium/components`, especialmente para flujos multi paso.

### http-bot

**Para que sirve:** exponer estado del bot por API HTTP.

**Como usarlo:** mira `status.controller.ts`, `status-http.service.ts` y `config/http.config.ts`.

### i18n-bot

**Para que sirve:** internacionalizacion por usuario y comandos de locale.

**Como usarlo:** estudia `greet` (traduccion por usuario) y `locale` (cambio y consulta de preferencia).

### schedule-bot

**Para que sirve:** jobs programados con `@spraxium/schedule`.

**Como usarlo:** revisa `startup-tasks.service.ts` para jobs one-shot y `scheduled-tasks.service.ts` para jobs recurrentes.

### signal-bot

**Para que sirve:** recibir eventos asincronos por webhook de Discord.

**Como usarlo:** empieza por `config/signal.config.ts` y luego revisa `signals/` y `schemas/`.

### sandbox

**Para que sirve:** referencia amplia con muchos modulos combinados en un solo bot.

**Como usarlo:** util para estudiar arquitectura de un proyecto mas grande y modular.

## Como ejecutar cualquier ejemplo

Desde el root del repositorio:

```bash
pnpm install
```

Luego, por ejemplo con `slash-bot`:

```bash
cp apps/slash-bot/.env.example apps/slash-bot/.env
# completa DISCORD_TOKEN en .env
cd apps/slash-bot
pnpm dev
```

> Los ejemplos estan orientados a aprendizaje y pruebas locales. Usalos como base y adapta seguridad, observabilidad, persistencia y deploy para produccion.
